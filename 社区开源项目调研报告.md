#### 可借鉴的 3 个核心算法

**1. 时间衰减函数**

```python
# Mem0 的记忆衰减公式
def decay_score(memory, current_time):
    age_days = (current_time - memory.created_at) / 86400
    access_boost = memory.access_count * 0.1  # 每次访问 +10%
    recency_boost = 1.0 / (1.0 + age_days * 0.05)  # 50天半衰期
    return recency_boost * (1.0 + access_boost)
```

```javascript
// Noe 可以在 NoeMemoryCurator 里加的衰减逻辑
function decayScore(memory, now = Date.now()) {
  const ageDays = (now - memory.createdAt) / 86400_000;
  const accessBoost = (memory.accessCount || 0) * 0.1;
  const recencyBoost = 1.0 / (1.0 + ageDays * 0.05);
  return recencyBoost * (1.0 + accessBoost);
}

// GC 时按 decayScore 排序，低于阈值的删除
function gcMemories(db, threshold = 0.2) {
  const memories = db.prepare("SELECT * FROM memories").all();
  const scored = memories.map(m => ({ ...m, score: decayScore(m) }));
  const toDelete = scored.filter(m => m.score < threshold);
  // ... 执行删除
}
```

**2. 语义冲突检测**

```python
# Mem0 的冲突检测：新记忆与已有记忆语义相似但事实矛盾
def detect_conflict(new_memory, existing_memories, threshold=0.85):
    for old in existing_memories:
        similarity = cosine_similarity(new_memory.embedding, old.embedding)
        if similarity > threshold:
            # 语义相似，检查是否事实矛盾
            if facts_contradict(new_memory.text, old.text):
                return ConflictResult(old=old, new=new_memory, type="contradiction")
    return None
```

```javascript
// Noe 已有 NOE_MEMORY_DEDUP_SEMANTIC，可以增强
// 当前：只合并重复，不处理矛盾
// 改进：检测矛盾并标记
function detectContradiction(newFact, existingFacts, threshold = 0.82) {
  for (const old of existingFacts) {
    if (old.similarity > threshold) {
      // 语义相似，检查事实方向
      if (factsContradict(newFact.text, old.text)) {
        return { type: 'contradiction', old, new: newFact };
      }
    }
  }
  return null;
}
```

**3. 记忆重要性评分**

```python
# Mem0 的重要性评分：综合多个维度
def importance_score(memory):
  score = 0.0
  # 1. 实体密度（提到多少人/地/事）
  score += len(extract_entities(memory.text)) * 0.1
  # 2. 情感强度
  score += sentiment_intensity(memory.text) * 0.2
  # 3. 信息量（TF-IDF）
  score += tfidf_score(memory.text) * 0.3
  # 4. 时效性
  score += recency_score(memory) * 0.4
  return min(1.0, score)
```

#### 集成方案

```
集成方式：借鉴算法，不引入 Mem0 依赖
原因：Mem0 是 Python，Noe 是 Node.js；引入 Python 依赖会增加复杂度

具体做法：
1. 在 NoeMemoryCurator 里加 decayScore() 函数
2. 在 NOE_MEMORY_DEDUP_SEMANTIC 逻辑里加矛盾检测
3. 在记忆写入时计算 importanceScore 并存储
4. 在 GC 时按 decayScore 排序淘汰
```

```
Letta 的"自我编辑记忆"：
┌─────────────────────────────────────────┐

│       ↓                                  │
```

#### 可借鉴的架构

```python
# Letta 的记忆管理 Agent（简化版）
class MemoryManager:
    def __init__(self, llm, memory_store):
        self.llm = llm
        self.store = memory_store

    def on_new_information(self, context, new_info):
        # 让 LLM 判断是否需要更新记忆
        prompt = f"""
        当前记忆：{self.store.get_core_memory()}
        新信息：{new_info}

        请判断：
        1. 是否需要更新核心记忆？(yes/no)
        2. 如果是，是替换(哪个)还是追加？
        3. 是否需要遗忘某些旧记忆？(哪些)

        输出 JSON: {{"action": "replace|append|ignore", "target": "...", "content": "..."}}
        """
        decision = self.llm.chat(prompt)

        if decision["action"] == "replace":
            self.store.replace(decision["target"], decision["content"])
        elif decision["action"] == "append":
            self.store.append(decision["content"])
```

#### 集成方案

```
集成方式：在 NoeMemoryCurator 里加 LLM 判断层
触发时机：每次 FactExtractor 提取到新事实后
LLM 选择：用本地 Ollama（零成本）判断是否重要

具体做法：
1. 新建 src/memory/NoeMemoryDecision.js
2. 每次 factExtractor 输出 → 调本地模型判断"是否值得记住"
3. 决定：append / replace / ignore
4. 在 NoeLoop 的 tick 里定期让 LLM 回顾记忆，决定遗忘
```

```typescript
interface Action {
```

```python
class KhojScheduler:
    def __init__(self):

        """定时执行 Agent 任务"""
        self.jobs.append({
            'cron': cron_expr,
            'func': agent_func,
            'kwargs': kwargs
```

```
集成方式：借鉴 The Swarm 的"代码自改自测"循环
原因：Noe 的自进化目前只限于"共识投票"，没有真正的代码修改能力

具体做法：
1. 在 ActPipeline 里加 executor: 'self_modify'
2. 流程：LLM 分析代码 → 生成 patch → 应用 → 跑测试 → 评估
3. 有完整回滚：git stash / git checkout
4. env 门控：NOE_SELF_MODIFY=1（高风险，默认 OFF）
5. 需要 owner 显式授权才能执行
```
