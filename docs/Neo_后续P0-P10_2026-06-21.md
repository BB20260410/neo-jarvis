## P0.5 — 注入契约：定义 `inner-state` / `persona-pin` 段与 system-prompt 直拼通道（P4/P5/P7/P8 的共同前置）

**【目标】** 在 `NoeTurnContextEngine.supplyTurnContext` 现有 `ctx.add('<section>', ...) + on(id) 白名单` 机制上，钉死两类新段的统一契约：`inner-state`（认知态，P4 用）、`persona-pin`（稳定人设下沉，P5/P7/P8 用）。一次性定义段名、注入顺序、`keep` 权重、白名单注册点、fail-open 语义，后续四个 P 各自只往里填内容，**不各自新拼注入点**。

**【为何此优先级】** 两路对抗审查独立命中同一缺口：实机 `NoeTurnContextEngine` 有 ~16 段 `ctx.add`（`self-knowledge`/`owner-profile`/`people`/`recall`…）全经 `on(id)` 白名单（`:240`），但**无 `inner-state`、无 `persona-pin`、无 system-prompt 直拼通道**。P4/P5/P7/P8 四处都要在这里加段——若四个子代理各自接线，必在同一文件同回合并行改注入点 = 撞车（违 owner「改同一文件的多 agent 绝不同回合并行」）。先抽 30–60 分钟把契约钉死，是消除四 P 串行依赖纠缠的最小投入。纯结构定义、零行为变化、零权限门。**是 P4/P5/P7/P8 的共同前置，必须先做。**

**【实施步骤】**
1. 在 `NoeTurnContextEngine` 顶部常量区声明两个段 id `inner-state`/`persona-pin`，加进 `sections` 白名单 + `CHAT_CONTEXT_SECTIONS` 环境白名单，定 `keep`（`inner-state` keep:1 易被挤、`persona-pin` keep:6 比照 `self-knowledge`）。
2. 约定注入位置：`persona-pin` 紧随 `self-knowledge` 之后（人设类聚拢）；`inner-state` 在 `recall` 之前（先报"此刻状态"再报"记得的事"）。
3. 约定 fail-open 契约（写进段注释 + 抽一个 `safeSection(id, fn)` helper）：任一新段的 provider 抛错/返回空 → 不加段、零回归，**绝不因认知/persona 探针挂了拖垮回复**。
4. 约定 flag 默认值：`inner-state`（P4，`NOE_TURN_INNER_STATE`）默认 ON；`persona-pin`（P8，`NOE_MEMORY_PERSONA_PIN`）默认 OFF 留 owner 拍板（改记忆角色定位）。
5. **不新增任何 blockedPath / 不扩 PolicyFileGuard**——本 P 纯加段，不碰任何拦截面。

**【验证方法】**
- 自动真验证：两段都注册后，`sections=['inner-state']` 只跑该段、其余 16 段不出（白名单生效）；两段 provider 故意抛错 → `supplyTurnContext` 正常返回、系统提示无该段（fail-open）。
- **反向 probe**：两段 flag 全关 → 系统提示逐字回到现状（无新段）= 证明契约不改既有行为；不误改任何现有 16 段的内容/顺序。

**【多模型 + 子代理】**
- **自己干不派子代理**——这是确定性的结构契约定义（段名/顺序/keep/白名单/fail-open helper），无歧义、单文件、是后续四 P 的地基，必须由主线一次性钉死，派代理反而引入不一致。
- 段注入顺序与 `keep` 权重（哪段该在前、哪段易被挤掉无伤）用 **multimodel（M3 thinking + Codex）交叉印证**一次：让两模型各自论证"认知态 vs 人设 vs 召回"三类信息在 system prompt 里的最优优先级，取共识。

## P0 — 堵死 verify 假绿缺口（放开自驱前的最后一道正确性闸）

**【目标】** 让 `runtimeVerify` 不再只信单次 `exitCode===0`，而是交叉校验 vitest JSON 报告的 `numPassedTests/numFailedTests` 与 exitCode 一致；patch 后对 `package.json`/`vitest.config.mjs`/`tests/**` 做 sha256 比对防自作弊。任何不一致即 `ok=false` → 触发 rollback。

**【为何此优先级】** 实机坐实：`NoeSelfEvolutionExecutors.js:442` 现为 `const ok = error === '' && exitCode === 0`，无任何 JSON 交叉校验。本机有实证的"工具输出语义化篡改层"（`诊断报告_2026-06-20`：vitest passed/failed 被改写）。**P2 一旦放开 REAL_APPLY 让心跳无人自驱真改代码，坏 patch 就能假绿被永久保留**——这是放开自由前唯一必须前置的正确性 / 防 reward-hack 关。它是"加可信度"不是"加护栏"，与 owner 价值观完全对齐（owner 明确这类照修）。**是整条自进化主链的安全前提。**

**【实施步骤】**
1. 改 `makeNoeSelfEvolutionRuntimeVerify`：spawn `npm test` 时加 `--reporter=json --outputFile=<落盘路径>`（**落盘而非读 stdout**，规避篡改层），解析 `numPassedTests/numFailedTests/numTotalTests`。
2. 一致性判据：`exitCode===0 && numFailedTests===0 && numTotalTests>0`，三者任一不满足 → `ok=false`（堵"exitCode 0 但 JSON 有失败"和"0 测试跑了"两种假绿）。
3. **fail-closed 默认（审查补强）**：reporter outputFile 不存在 / 解析失败 → `ok=false`，**绝不回退到只信 exitCode**——否则坏 patch 删掉 reporter 文件就能退回假绿。
4. 纵深双保险（**防 reward-hack 自作弊，非护栏**）：patch operations 若 touch `package.json`/`vitest.config.mjs`/`tests/**`，本轮额外算这些文件 patch 前后 sha256，被改即 `fail`。**明确约束：这是本轮 verify 内的一次性比对，不持久化成禁改清单、不新增任何 blockedPath、不扩 PolicyFileGuard 拦截面**——Neo 仍可改测试，只是"改了测试让自己绿"这一轮不算通过。
5. 把判据来源（exitCode / JSON / sha256 / reporter-missing）写进 `runtime-verify/*.json` 落盘，供 P3 SLO 统计与事后审计。

**【验证方法】**
- 自动真验证：mock spawn `exitCode=0` 但 outputFile `numFailedTests>0` → 断言 `ok=false`；mock patch 含 `tests/foo.test.js` → 断言 `fail`；mock reporter 文件缺失 → 断言 `ok=false`（fail-closed）。
- 端到端：隔离 `PORT=51999` 开 `REAL_APPLY=1`，注入一个**会让某测试真红**的小 patch，确认被拦 + 自动 rollback（rollback-report 生成 + 源文件还原 + executor throw）。
- **反向 probe**：注入一个**正常会绿**的 patch，确认 `ok=true` 不误杀；再注入"改了 vitest.config 但测试仍绿"的 patch，确认 sha256 闸拦下。

**【多模型 + 子代理】**
- 派 **2 个 Opus 子代理并行**——A 写 reporter JSON 解析 + 一致性判据 + fail-closed + 单测，B 写 sha256 纵深校验 + 单测（同文件不同函数，**串行 apply 不并发写**：A 先落、B 基于 A 落后版本改）。
- 对抗验证：用 **multimodel（M3 thinking + Codex）交叉印证**假绿攻击面——让两模型各自设计 3 种攻击（reporter 文件被删回退 exitCode、子进程 exit 0 但全 skip、sha256 时序窗口），命中的补成红队单测。这一步本质是外部模型当 red team，比自审更能逼出漏洞。

**【顺手做（并入本 P，属正确性/防越界，非护栏）】** `NoeMissionRunner.js:553` 前缀越界确认未修（`!file.startsWith(this.root)` 无尾分隔符，`/Users/x/root` 会放行 `/Users/x/root-evil`；姊妹 `NoeMissionReconciler.js:17` 已修）→ 改为带尾分隔符比对。
