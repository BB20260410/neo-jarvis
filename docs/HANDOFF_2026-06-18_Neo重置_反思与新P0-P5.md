## 1. ⭐ 反思：这几天重复空耗的根本病

**模式（每天一样）**：`发现空转点 → 建/修一个机制 → 点火 → 报喜 → 次日发现还没用 → 再发现新空转点`。

几天建了十几个认知机制（议会深思 / 自我进化 / 梦境 / 好奇回路 / 定时调度 / 动态选题 / 照妖镜 / 期望账本 / GWT / 情感 VAD…），**全部「机制活着、产出空」**。

**病根：把一个「闭环能不能闭合」的二元问题，做成了「建很多机制」的系统工程。**
- 建机制有即时成就感（代码 / 测试绿 / commit / "已打火"），会上瘾；
- 但闭环（学到 → 召回 → 用上 → 改行为）一天不闭合，机制再多 = 0；
- 5202 测全绿，**没有一个证明「Neo 因学习而变好」——一直在度量错的东西**；
- 连本窗最初写的那份 P4-P10 交接（已删），都还是「再建 7 个机制」的同一个病。

**三误区（写给下一个接手的自己，别再犯）**：① 机制建好 ≠ 功能生效；② 攻击顺序反了（一直加固「搜得更好」的前半段，断点在「学进行为」的最后一公里）；③ 没有成效照妖镜 → 注定感受不到成效（本窗已补）。

## 2. 新思路 + 行为整改（钉死）
1. **一个真闭环 > 十个半成品**。停止建任何新认知能力，只闭合一个 owner 能亲眼验证的最小学习环。
2. **成效锚定「对 owner 直接有用」**，不是「Neo 在后台变聪明」（后者天然不可见、反馈以月计）。
3. **科研归科研**：自我意识/GWT/情感/梦境/自进化 = 后台实验，代码留着不删，但**不占主线、不作为成效来源**。
4. 完成标准 = owner 可验证的闭环成效，**不是「测试绿/已打火」**；动手前先问「**这服务于闭合哪个环？不服务就不做**」；每个 milestone 给 owner 看**真 demo / 照妖镜数字**，不汇报「改了什么」；**敢砍**（技术有趣 ≠ 对 owner 有用）。

## 5. ⭐ 新 P0-P5（以闭环 + 对 owner 有用为中心）

> 原则：**不建新机制，只闭环 + 转向对 owner 有用。每个任务做完必跑照妖镜验收。**

### P0 止血止耗 —— 先别再挖坑
- 做：把高频空转的算力消耗降下来——`continuousLearning` 立 self_learning 的频率调低（现 295次/天）、自进化/好奇空转循环降级或暂停。闭环没通前学得越勤越浪费本地脑。
- 怎么做：`NoeGoalSystem` `learningIntervalMs` 调大 / `continuousLearning` 加「上一轮 lesson 未被召回则退避」闸；或直接把 self_learning 自动立项频率压到每天个位数。
- 验收：照妖镜「共学次数」增速明显下降，本地脑（LM Studio）空闲占比上升，**且不影响 owner 主动发起的学习**。

### P1 闭一个能 demo 的最小学习环 —— 本窗后最高优先
- 做：选 **1 个**具体知识点，端到端跑通 `搜 → 读 → 提炼成 1 条可召回 lesson → 下次遇相关问题真召回用上`。
- 关键修（皆已有定位，别重新建）：① learning goal 的 act 步因 `NOE_GOAL_ACT≠1` 降级 think→goal 永 open，点火 `NOE_GOAL_ACT`（owner 在场）；② think 末步真深思产 lesson（`NOE_THINK_DELIBERATE` 已落地未点火，`c761637`）；③ 召回纳入 lesson（`NoeDeliberation.js:90` recall 加 `sourceType:['skill_distill','surprise_lesson']`+`bumpHits`）。
- 验收（照妖镜 + 人工）：这条 lesson hit_count≥1 **且** 人工确认它改了某次具体回答/行为。**把「学习有用」从口号变成一个能演示的事实——这一条做成，胜过过去几天全部。**

### P2 学习对话可见 —— 让 owner 感受到
- 做：Neo 跟 owner 对话时，真召回学过的 lesson 答得更好（聊天主链 RAG 纳入 skill_distill/surprise_lesson 池）。
- 验收：owner 问一个与已学主题相关的问题，回答里能追溯到具体 lesson（可在透视页/日志看到召回来源）。

### P3 转向「学 owner」而非「学网页」 —— 最快可见的价值
- 做：把自主学习的重心从「上网学 AI agent 知识」转向「记住 owner 的偏好/习惯/项目上下文并真用上」。owner 的偏好已有沉淀（记忆库 user/feedback 类），让 Neo 主动召回并应用在对话/主动陪伴里。
- 验收：Neo 在新对话里主动体现 owner 已知偏好（如语言/工作方式/项目焦点），owner 能感受到「它真的懂我、记得我」。

### P4 召回质量地基（原 P9）—— 撑起 P1-P3 的「用上」
- 做：根治语义召回失效（project/insight 探针近乎 0）、热点垄断（top8 占 92% hit）、向量盲区（离线写入无 embedding）。

- 验收：insight/project 召回命中率从 ~0 涨；照妖镜③被用率 19%→≥40%。

### P5 成效证伪（原 P10）—— 让成效彻底可证伪
