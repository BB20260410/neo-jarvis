# 社区项目对标矩阵与后续路线 2026-06-11

> 本文是 Claude Fable 5 中断搜索任务的完整收口版：把已搜到和补搜到的官方项目资料转成 Noe 可执行路线。星数、热度会变，本文不把它们作为决策依据；以官方仓库/论文/文档定位、Noe 当前代码事实、集成风险和验证门作为依据。

## 1. 总判定

Noe 当前不是缺“再装一个 agent 框架”，而是缺三类工程闭环：

1. **本地持续运行闭环**：模型可用性、语音链路、守护状态、故障自诊断必须进入可见/可修复状态。
2. **目标到行动闭环**：owner 委托、Noe 自生目标、研究、只读诊断、审批等待、执行结果、复盘要形成一条可恢复链。
3. **记忆到人格闭环**：事实更新、冲突解决、时间更替、睡眠整理、长期偏好和行为反馈要变成可评估数据。

社区项目的价值是“借工程方法”，不是把 Noe 迁到 Python/云端/大框架。

## 3. 对 Claude 草稿的纠偏

`社区开源项目调研报告.md` 可以作为原始搜索材料，但不能直接当实施计划：

- “升级 qwen2.5:32b 替代 gemma3:4b”不是当前裁决。模型选择必须以 `doctor:noe:models`、LM Studio/Ollama 当前可用模型、内存压力和实际中文/工具调用评测为准。
- “launchd 守护无”已不准确。当前 51835 live 有既定启动/重启边界；系统级调度或生产重启必须按 owner 授权和端口检查执行。
- “OpenMeow 直接替换语音链路”风险过大。Noe 已有 Sherpa/Kokoro/CosyVoice，OpenMeow 只适合作为可选兼容网关。
- “内心独白直接触发行动”不采纳。Noe 必须保持 act 明示声明、目标系统、ActPipeline、审批/审计这条链。
- 大框架迁移不是第一优先级。Noe 的优势是本地治理、权限、安全、记忆和 owner dogfood，不应为了 LangGraph/CrewAI 等框架推倒现有系统。

## 4. 完整后续执行路线

### P0：完成 Claude 中断任务的工程闭环

目标：让 owner 交办“去找原因/排查本地问题”时，Noe 真的读本地项目证据，再归因。

已落地：

- `NoeDelegationExtractor` 区分外部资料型 research 与本地排查型 act。
- 本地排查型委托先跑只读 `rg`，再把输出交给后续 think。
- act 输出压缩进 goal step note，后续 step 可读 `priorNotes`。
- 语音续播失败重试、迟到续播接上、CosyVoice 兜底已覆盖。

下一步可继续做，但不再属于 Claude 中断任务阻塞项：

1. 给常见本地域加更窄诊断模板：voice/model/memory/goal/panel。
2. 把 act stdout/stderr 摘要从 goal note 同步到 Activity 明细。
3. 给 owner 透视页目标卡展示“诊断证据”折叠块。

验收：

- owner 说“语音系统又出问题了，你去找原因”后，goal plan 至少包含一个只读 act 步和一个归因 think 步。
- act 步完成后，下一步能看到 `stdout:` 证据。
- 被安全门阻断时不重复抢占、不伪装完成。

### P1：记忆升级线（Mem0 / Letta / Graphiti / Zep）

目标：把“记得住”升级为“会更新、会遗忘、会处理变化”。

实施顺序：

1. 新增记忆评测 fixtures：偏好更替、事实冲突、旧事实失效、同义重复、低盐度噪声。
2. 在 `NoeMemoryCurator` 或 MemoryCore 旁边加纯函数 policy：`merge | supersede | keep_both | ignore | needs_review`。
3. FactExtractor 输出增加 `validFrom/validTo/confidence/sourceEpisodeId`。
4. 把 Graphiti/Zep 的 temporal knowledge graph 思路映射到现有 NoeKnowledgeGraph，不跑外部服务。

对应文件：

- `src/memory/FactExtractor.js`
- `src/memory/NoeMemoryCurator*`
- `src/knowledge/*`
- `tests/unit/noe-fact-extractor.test.js`
- 新增 `tests/unit/noe-memory-conflict-policy.test.js`

验收：

- “我喜欢美式”后又说“现在改喝拿铁”，查询当前偏好只能返回拿铁，并保留美式历史。
- 同义重复不会膨胀记忆库。
- salience 高的 owner 明确事实不能被夜反思误删。

### P2：目标执行状态机线（LangGraph / OpenHands / Hermes）

目标：目标执行可恢复、可解释、可暂停，不靠一轮模型输出赌结果。

实施顺序：

1. 给 goal step 增加显式状态原因：`open/doing/awaiting_approval/blocked/done`。
2. 把 `research_started/research_done/act_started/act_done` 串成可查询 ledger。
3. checkpoint 包含：goal id、step index、input、tool/action、policy decision、output digest。
4. 恢复逻辑只恢复安全状态：awaiting approval 不自动重放危险命令。

对应文件：

- `src/cognition/NoeGoalSystem.js`
- `src/cognition/NoeWorkspace.js`
- `src/loop/ActPipeline.js`
- `src/audit/ActivityLog.js`
- `tests/unit/noe-workspace-goals.test.js`

验收：

- 后台中断后，doing step 可被标记 recovered/failed，而不是永久卡死。
- approval 等待不重复生成新审批。
- 每个目标完成都有可审计证据链。

### P3：语音网关线（OpenMeow）

目标：在现有语音链路稳定后，增加可选 OpenAI-compatible 本地语音网关。

实施顺序：

1. 新增 `OpenAICompatibleVoiceGatewayClient`，只接受注入 base URL，默认 OFF。
2. 只在 `NOE_VOICE_GATEWAY=1` 时接入 `_synthesize` fallback 链。
3. 保留 CosyVoice；OpenMeow 不可用时不影响现有链路。
4. 增加 doctor 检查：服务可达、voices 列表、一次短句合成。

验收：

- 无 env 时行为零差异。
- gateway 不可达时 fallback 到 CosyVoice 或返回清晰错误。
- 不读取或打印任何 secret。

### P4：浏览器/GUI 操作线（UI-TARS / browser-use / OpenHands）

目标：从“看屏幕”升级到“可预演的受控操作”，但不绕过 owner 审批。

实施顺序：

1. 先做 `visualActionPlan`：截图/DOM -> 计划，不执行。
2. 每个动作进入 ActPipeline，显示 target、risk、rollback/evidence。
3. owner 批准后才执行单步；默认只允许浏览器安全动作，桌面全局动作继续 blocked。
4. 逐步接入 Browser/Computer Use 工具，不自造高权限点击器。

验收：

- 模型能生成“要点击什么/为什么/风险”而不是直接点。
- 危险动作必须进 approval。
- 截图、DOM 证据和执行结果进入 Activity。

### P5：入站渠道与守护线（QwenPaw / OpenClaw / Hermes / Khoj）

目标：Noe 能被稳定唤醒、接收外部事件、记录来源，并在边界内响应。

实施顺序：

1. 先做单进程 EventBus 抽象：source/channel/user/message/permissions。
2. Telegram 或本地 webhook 作为第一个受控入站，mention-gating + allowlist。
3. 所有入站写 Activity 和 timeline，默认不触发高风险 act。
4. 守护/launchd 只提供用户主动安装脚本和 doctor，不在 Codex 自动安装。

验收：

- 入站消息能形成 owner_interaction 或 external_event。
- 未授权来源不触发目标/act。
- 所有入站有 source id 和审计 trace。

### P6：自我改进研究线（DGM / SEAL / Reflexion）

目标：保持 Noe 可自我改善，但只在现有共识和测试门下运行。

实施顺序：

1. 每次自改生成 variant archive：变更原因、diff、验证、回滚方式。
2. 自主性指标不退：目标完成率、接地率、Brier、语音完成率。
3. SFT/LoRA 只等数据量够再做；不拿少量对话强训人格。
4. Reflexion 类机制只写 retrospective，不直接授权执行。

验收：

- `npm test`、`verify:noe:self-evolution`、目标级 completion audit 全绿。
- 有 rollback 证据。
- 没有 passed ledger 不写长期记忆授权。

## 5. 下一窗口执行顺序

如果下个窗口继续开发，按这个顺序：

1. 先跑 `npm run doctor:noe:models` 和 `npm run noe:keys:model:check`，只看状态不看密钥。
2. 不要继续扩大社区搜索；直接选 P1 记忆升级或 P2 目标状态机做第一刀。
3. 每一刀都必须小范围改文件、补单测、跑 `npm run test:p0:unit` 与相关定向测试。
4. 若涉及 live 51835，先 `lsof -nP -iTCP:51835 -sTCP:LISTEN` 确认归属，再按 owner 授权重启；永远不碰 51735。
5. 若涉及真实外部发布、Obsidian、Telegram、桌面点击、浏览器登录态，先停在 approval/blocked，不代替 owner 授权。
