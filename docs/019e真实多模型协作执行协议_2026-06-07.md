## 真实调用入口

在真实仓库执行：

```bash
cd "/Users/hxx/Desktop/Neo 贾维斯"

npm run noe:consensus:round -- \
  --goal "用真实多模型协作完成当前任务的方案与第一实现切片" \
  --evidence-text "把本轮用户目标、相关文件、当前 git 状态、硬边界、验收目标写清楚" \
  --round-id "019e-training-$(date +%Y%m%d%H%M%S)" \
  --run-models \
  --ack-cost
```

该命令必须产生：

- `output/noe-multimodel/<round>/brief.md`
- `output/noe-multimodel/<round>/manifest.json`
- `output/noe-multimodel/<round>/codex.txt`
- `output/noe-multimodel/<round>/claude.txt`
- `output/noe-multimodel/<round>/gemini.txt`
- `output/noe-multimodel/<round>/m3.txt`
- `output/noe-multimodel/<round>/ledger.json`

然后验证：

```bash
node scripts/noe-consensus-ledger-verify.mjs \
  --ledger "output/noe-multimodel/<round>/ledger.json" \
  --require-passed
```

如果 `--require-passed` 不通过，不能进入实现；必须说明 blocker。常见 blocker：

- 某模型没有真实 raw output。
- 只剩一个模型可用。
- approval 数量不够动态 quorum。
- M3 输出包含执行/写入内容，违反 suggestion-only。
- Claude/Gemini 越权写入。
- 缺同一 evidenceRef。

## 实现任务的固定流程

1. `root_check`：确认 `/Users/hxx/Desktop/Neo 贾维斯` 是真实 git repo。
2. `evidence_brief`：把用户目标、相关文件、当前 git 状态、边界、验收命令写成同一证据。
3. `real_model_round`：运行 `npm run noe:consensus:round -- --run-models --ack-cost ...`。
4. `ledger_gate`：对目标 ledger 跑 `--require-passed`。
5. `codex_implementation`：只有 gate passed 后，Codex 做最小实现。
6. `verification`：跑目标单测、脚本验证、必要的接口/UI smoke。
7. `post_review`：Claude/Gemini/M3 对 diff/test/report 做非实现复审；不可用要有 rawOutputRef。
8. `cycle_or_audit`：需要闭环时组装 cycle artifact 或跑 completion audit。
9. `final_report`：只能报告已经被证据证明的完成项。
