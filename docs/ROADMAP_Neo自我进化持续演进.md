# ROADMAP · Neo 自我进化的持续演进

> 一份**活**的路线图:不是一次性清单,而是一个有节奏、以证据为门、会自我剪枝的循环。每次复盘按真实数据重排优先级、砍掉没用的、推进被证明有效的。
> 制定:2026-07-03。载体:本文件长期演化(改这里,不新建)。

## 唯一原则:先能测,再变强,最后自己决定怎么变强——每步都要上一步的真实数据证明有效才推进

这是把科学方法用在自我进化上。**测量在最前面,因为没有一把超越"绿测试"的尺子,所有进化都是漂移不是进步**(飞轮曾在 68% 回滚率下自report"健康")。这也对齐 owner 的铁律:禁假数据、真验证。

配套的元原则(让这份计划"不断改善"):它是一个**带节奏、以证据为门、会自我剪枝的循环**——每阶段只在上一阶段的指标曲线真动了才推进;指标证明没用的项,砍掉。

## 阶段一:让飞轮"看得见自己是否变强"(测量——一切的前提)

**没有这个,后面全是盲调。这是北极星仪器。**

1. **能力题库(ground-truth battery)**:一套 held-out 的真实自改任务(不同难度:JSDoc/小逻辑/跨模块重构/修真 bug),Neo 定期重跑,量"通过率随时间"。这是唯一能回答"Neo 真的更能干了吗"的尺子。绿测试只答"没弄坏",答不了"变强了"。
2. **诚实仪表盘**:真进步率(真保留改逻辑/总尝试)、回滚归因分布(verify/双绿门/护栏各占多少——reason 已通电,现在只差聚合视图)、教训→避免重复的命中率、本地 vs 云端 implement 占比、能力地图覆盖。
3. **门**:阶段二/三的每一项,都要能在这仪表盘上看到它移动了曲线,否则砍掉。

**我可自主驱动**(全只读+落盘,零风险)。

## 阶段二:让飞轮"改得动更难的东西"(扩前沿)

1. **难目标分解**:对本地模型一把梭会失败的大改动,先跑一个"规划 pass",拆成一串**每步都结构化、本地可做**的微改动。把"不可能"变成"一串可能"——人重构就是这么做的。这是本地进化能力的最大杠杆。
2. **逃离琐碎最优(探索预算)**:刻意留一部分 cycle 去啃高价值/低成功率的硬目标,别永远种 JSDoc。像 RL 的 explore vs exploit。配合阶段一的能力地图:知道自己能做什么后,才能有意识地去够不着的地方。
3. **能力地图显式化**:把轴4 攒的失败教训合成一个 Neo 立项前能读的自我模型——"改文件Y内部逻辑我会失败",据此预测"我能不能成"、决定本地做/云端做/先分解。
4. **知识图谱当推理底座**:我已把 KG 接进召回;下一步让它支撑**架构级**目标(这是 god-object、这三个耦合了),而不只是行级 smell。非琐碎目标的来源。

**我可自主驱动 1/2/3**(flag 默认 OFF,隔离验证再点火);4 是中等工程,分步。

## 阶段三:让飞轮"进化自己的进化策略"(元自适应)

1. **MetaEvolution 做策略实验**:提出可 A/B 的进化策略实验——"两天内 self_directed 权重×2、test_gap×0.5,量真进步率,更好就采纳"。给自己的进化政策做对照实验。**采纳仍 owner 门控**(advisory→提议实验,不自动改)。
2. **蒸馏技能闭环**:learn→判真伪→启用→度量有没有帮上忙→留或删。治"100 技能全 disabled"——学了不用等于日记不是成长。
3. **整合与遗忘**:把很多情景压成原则、忘掉噪声(reject 教训现在是模板噪声)。真正的完善靠压缩,不靠堆。

**门控**:1 的实验采纳、任何碰策略权重的,都 owner 拍板;2/3 我可自主做闭环逻辑。

## 让这份计划自我改善的机制(这才是"不断完善"的关键)

1. **复盘节奏**:定期(建议每周或每积累 N 个进化样本)做一次自我复盘——读仪表盘,问三个问题:①当前最大瓶颈是什么?②这轮学到了什么?③下一个最高杠杆是哪件?据此重排本文件的优先级。计划是循环,不是清单。
2. **证据为门**:每阶段只在上一阶段指标真动了才推进。别在阶段二没抬起能力曲线时就建阶段三。
3. **自我剪枝**:指标证明没用的项,从本路线图删掉,写明"实测无效,已砍"。诚实地缩减 > 假装全做。
4. **谁驱动**:安全+可逆的我自主做(你已授权点火);碰安全/调参/对外的,我提议+你一句话决定;我每轮把"改了什么、指标动没动、下一步"两三句报你,不复读 diff。

## 执行记录 · 阶段二已启动(2026-07-03)

按仪表盘证据(非拍脑袋)推进,已落地并点火:
- **Step1 自我复盘心跳**(NOE_EVOLUTION_REVIEW=1):每天读 panel.db 落一份仪表盘快照到 history.jsonl,真进步率趋势曲线**自动累积**——「每天都比昨天更强」从此可测、循环随时间自转。
- **阶段二A 探索广度**(NOE_SELFEVO_SIGNAL_WEIGHTING=1):按信号历史保留率加权选目标——治 test_gap 黑洞。真实数据验证权重:test_gap 0.61(降权)、self_directed 0.88/high_complexity 0.85(最高)、地板 0.35 不饿死任何信号(保探索广度)。转投高保留信号 = 真进步率该往上抬。
- **阶段二B 提高落地率**(NOE_SELFEVO_FROM_REASK=1):检测本地模型编造的 from(题库证实 fuzzy 都救不了)→ 附纠正提示 re-ask 一次(逐字复制/改 write_file)。端到端测试:编造→re-ask→逐字纠正→接受。

**证据为门(下轮复盘验证)**:这三项是否真有效,看 review 脚本的趋势——真进步率随时间升 = 有效保留;平/降 = 该调或砍。**这正是路线图自己的原则:用趋势说话,不是上线就算赢。**

**诚实证据(2B 首验未通过,不粉饰)**:re-ask ON 重跑题库,落地率**仍 50%、easy 仍 0%——没抬起**。re-ask 机制单测验证正确(编造→纠正→逐字接受),但在这 6 个确定性题库任务上没生效(本地模型 re-ask 后可能仍编造,或这两个 easy fixture 的失败非 re-ask 能治)。**结论:2B 机制对、低风险、flag 可控,但题库上未被证明有效**——保留观察生产多样目标的趋势(复盘心跳每天累积),若真进步率不动就按自我剪枝原则调/砍。这次是仪器把我拦住、没让我把"上线"当"有效"——正是它存在的意义。下轮该做:给题库补更有代表性的 easy 任务(2 个 fixture 样本太小),或换用"分解成小步"根治编造。

- **更智能的活动陪伴——懂什么时候该安静**(NOE_PROACTIVE_SMART_GATE=1):shouldSpeakProactively 超越时钟静默——主人正在专注工作(专注窗口内刚有互动)时不开口打断(即使白天),顺延到主人离开一阵再说;没真值得说的不闲聊;紧急事压过一切。这是「懂你什么时候需要、什么时候该安静」的第一块。
