# Noe 急需补强 · Top 100 开源选型（按优先级 + 互补簇排行）

> 日期：2026-06-10　｜　已排除：安全/沙箱/权限治理类（与路线图"不再建安全墙"一致）
> 排序依据 = 3 路代码诊断得出的真实短板（见下表），不是 star 高低。
> 选取标准：① 直击 Noe 当前短板 ② 集成成本（OpenAI/MCP/REST 兼容优先）③ 能与已有/彼此互补成栈。

## 诊断结论：Noe 的病根是"建好不通电" + 几处真空白

| 层 | 已有（勿重复造） | 真短板 |
|---|---|---|
| 语音输入 | mlx-whisper turbo（整段 POST，3-7s） | 无 VAD 端点 / 无 always-on 唤醒词 / 非流式 |
| 语音输出 | MiniMax 在线 + Kokoro(仅英文) | MiniMax 断网中文变哑 / 非流式启播慢 |
| 视觉 | qwen-3-vl-8b(LM Studio,5s/张) | 无 OCR / 无 UI 元素坐标 / 无后台轮询 |
| 执行 | shell 白名单38 + AppleScript/JXA + 社交DOM配方 | 浏览器无 Playwright/无截图验证 / 无 AX 树(cua 没接) / shell 无管道 |
| MCP | McpClientManager(能挂三传输) | 只挂 1 个且未运行，生态=空壳 |
| 记忆 | SQLite+FTS5 | 嵌入默认 128 维 hashing 零精度 / FusionRanker 零接线 / 图谱无时序 |
| 多模型 | room 12 adapter + BrainRouter(关键词) | 无统一网关 / 路由精度低 |
| 入站 | InboundChannels(库就绪) | Telegram 从未挂进 server，离机不可达 |
| 主动陪伴 | proactiveTick(逻辑全) | 无进程内定时触发，靠前端戳 |

## 没进前 100 的（及原因）

- **安全/沙箱/权限治理类**（按你要求 + 路线图"不再建安全墙"剔除）：toolhive(MCP 容器隔离)、bytebot(隔离执行)、Helicone/vector-admin(偏管理)、各类 guardrail。
