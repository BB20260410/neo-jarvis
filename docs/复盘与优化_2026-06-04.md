## 二、延迟实测与瓶颈

实测（2026-06-04，本地 panel，abliterated:4b 已加载）：

| 环节 | 实测耗时 | 是否瓶颈 |
|---|---|---|
| whisper STT（本地） | **0.46s** | 否 |
| 大脑生成（本地 abliterated:4b） | **2.6 ～ 4.5s** | **是（最大头）** |
| MiniMax TTS（云） | **~1.9s** | 是 |
| 前端 VAD 判停（silenceMs） | 0.9s | 固定开销 |
| 全链路（STT+大脑+TTS） | **~5s** | |
| 用户体感（说完→听到回应） | **~5～6s** | |

**结论**：
- `≤200ms` 是音视频**管线**（采集→显示）指标，当前前端 getUserMedia+canvas 推帧本就实时；与「AI 对话响应」是两回事。
- AI 首响现实 ~5-6s，**物理下限就在那**（LLM 生成 + TTS 合成）。
- 额外开销：ollama 在 abliterated:4b（对话）/ gemma4:31b（事实提炼，20GB）/ qwen-vl（视觉）之间**来回加载**，首次或切换 +数十秒（用户偶发"卡一下"的来源）。

**流式优化路径（目标：<2s 听到第一字）**，按收益排序：
1. **流式 LLM + 边生成边 TTS**（收益最大）：大脑流式输出，凑够一句立刻送 TTS 播放，不等整段生成完。首字节可从 ~4.5s 降到 ~1.5s。
2. **本地中文 TTS 替云**：省掉 MiniMax 1.9s 网络往返。当前 Kokoro 中文弱，需选型本地中文 TTS。
3. **模型常驻**：ollama `keep_alive` 拉长 + FactExtractor 换小模型（别让 20GB 的 gemma4:31b 挤掉对话模型），消除切换卡顿。
4. VAD 判停 0.9s→0.6s（小收益，太短会切断）；流式 STT（STT 才 0.46s，收益小）。

## 三、长期记忆问题（三层定位 —— 你最关心的）

| 层 | 状态 | 证据 |
|---|---|---|
| 后端持久化 | ✅ **在写** | panel.db `noe_memory`：voice 58 / proactive 13 / fact 5 / vision 1 条 |
| 前端显示 | ⚠️ 刷新丢 | 刷新后聊天记录消失（前端状态，未从后端加载历史） |
| **对话调用** | ❌ **真根因** | `VoiceSession._respond` 喂给大脑的 messages = `[system, 当前user]`，grep `recall/history/conversation` 全空 |

**根本原因**：Noe 的语音/文字对话是**「无记忆单轮」**——每句只看当前这一句，**不带历史、不 recall 长期记忆**（写进数据库的 voice/fact 记忆**只写不读**）。所以"刷新就忘 / 记不住"是「前端没加载历史」+「对话层本就不用历史」双重叠加，**不是后端真丢数据**。

**修复方向（第二阶段）**：① _respond 注入最近 N 轮对话（取 noe_memory voice scope）；② query 命中相关 fact 时 recall 注入（mem0/向量召回）；③ 前端刷新从后端拉历史渲染。

## 四、视觉能力现状

- **能力**：✅ 看屏幕（截屏→qwen-vl）+ 看摄像头（前端推帧→VLM），实测准确（"用户在用终端调试…""坐在车里看手机"）。
- **限制**：**单帧静态快照**，非连续行为流。VLM 看一张图描述"此刻像在做什么"，**没有动作序列 / 行为识别 / 事件检测**。
- **性能**：VLM 单帧 ~3s。本会话给 camera 每句 glance 加了 **3s 超时兜底**防卡。
- camera 无帧自动回退 screen（本会话修）。

## 五、主动陪伴现状

- **机制**：proactiveTick 看一眼（屏幕/摄像头）→ 本地大脑判断要不要开口 → TTS。
- **本会话已改**：提示词放宽（更愿开口）+ 冷却 1min（.env）+ camera 无帧回退。
- **仍不智能的点**：
  1. 靠 VLM **单帧描述 + 大脑单次判断**，无连续观察/行为理解（看快照，不知你"持续在做什么"）；
  2. **无记忆延续**（同第三节，不记得之前说过啥、你的偏好）→ 可能重复、不贴心；
  3. no_change 检测：屏幕不变就沉默（静态工作时哑火）；
  4. 无主动话题策略（只会"看到 X 说一句"，不会发起有意义的陪伴话题）。

## 六、根源分析（架构 / 模型 / 工程）

**架构**
- **无状态对话层**：VoiceSession / proactiveTick 都不带历史、不 recall，记忆只写不读 → 失忆根源。
- **视觉是单帧快照非流**：每次截一张图，无连续帧/事件流 → 行为分析浅。
- 对话/视觉/记忆三子系统割裂（本会话刚把**视觉**接进对话，**记忆**还没接）。

**模型**
- 本地大脑 abliterated:4b 生成 ~2.6s（4b 是速度/质量平衡点）；VLM ~3s；TTS 云 ~1.9s。
- 多模型挤一个 ollama（4b 对话 / 31b 提炼 / vl 视觉）来回加载，切换开销大。

**工程**
- **级联管线串行**：STT→大脑→TTS 顺序等待，非流式 → 首响 = 各环节之和。
- 无模型常驻策略；前端 VAD 判停固定 0.9s。
