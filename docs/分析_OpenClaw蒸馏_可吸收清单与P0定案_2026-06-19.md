## 一、两边架构本质
- **OpenClaw = 工业级多渠道 Agent 网关平台（横向广度）**：23+ 渠道插件 SDK、Gateway RBAC/配对/node-host 分布式、媒体生成全家桶（图/视频/音乐）、speech-to-speech 实时语音栈、ACP/IDE 桥、ClawHub skill 市场+签名供应链。威胁模型=对外网关暴露面+多操作员。
- **Neo = 单用户本机认知型个人 OS（纵向深度）**：主动环境感知（VisionSession）、级联语音首句早鸟、持久心跳+崩溃恢复+欠账补跑、SQLite 持久审批、权限治理纵深、自学习 skill、认知/意识层（梦境/议会/目标系统/GWT）。威胁模型=本机其他进程 curl（仅 127.0.0.1+owner-token）。
- **结论**：OpenClaw 横向"平台/多渠道"能力对单用户 Neo 多为**负债**（战略文档已四票否决多渠道网关）。Neo 在危险命令规则/SQLite 持久审批/权限引擎/secret 脱敏上**已强于或持平 OpenClaw**。真正可吸收的只剩"安全抓取硬化、运行态可观测、内容安全扫描"几块**小而硬**工程件——按 Neo 架构（Node.js + @ts-check + DI + 新文件<500行 + 单测 + .env flag 默认OFF + SQLite）重写，不照搬 TS 插件协议栈。

## 三、P0 定案（四方一致 + M3 补强）

1. **P0-2 verify 自检**（先跑，数据出来前优先）：量化 Neo 现有 verify 健康度——直接回应"不能只说源码存在"。通过率高则降 P1，低则修复破损反馈回路是真第一 P0。
2. **P0-1 SSRF 守卫统一**：抽 `src/security/SsrfGuard.js`（复用 img-cache 强版 isPrivateIp/assertPublicUrl + ipaddr.js 兜底字面量），WebSearch/PermissionGovernance/WebhookDispatcher/img-cache 四处统一 import，删两处弱版。论据=**owner 高频出站链路对 DNS rebinding 无防御 + 消除两套实现策略分裂**（非 Claude 被推翻的"四种写法"）。flag `NOE_SSRF_ALLOW_FAKEIP` 默认 OFF 留代理豁免。
3. **P0-3 链接自动理解**：先以"无 readability 纯抽取正文+摘要"落地，readability 作可选增强。

### 七·续：codex 六轮对抗式复核 + 51835 实测驱动的深度加固（同日续）
第一轮交付后经 codex 多轮读代码+探针复核 + 我自己 51835 实测，又挖出并修复一批更深的洞——这正是 owner"每步多模型验证+真实运行证据"要堵的"源码存在≠真安全"：
- **codex 抓的 9 个洞**：① `::7f00:1` IPv4-compat hex 真绕过（URL 把 `::127.0.0.1` 压成 hex，第一轮单测漏判）→ isPrivateIp 补"前96位为0的 `::x:y` 一律展开判私网" ② WebSearch/WebhookDispatcher TOCTOU 未闭合（只预检不绑连接层）→ 抽 `safeFetchPublicUrl` 统一入口（逐跳 assertPublicUrl + redirect:'manual' + pinned dispatcher） ③ DNS 预检无超时 → withTimeout 5s ④ timer 只覆盖响应头不覆盖 body 的 DoS → timer 收进 cleanup（caller 读完 body 才清） ⑤ webhook redirect 跨域转发 Authorization/secret → maxRedirects:0 ⑥ createPinnedLookup mismatch 回退裸 DNS → fail-closed ⑦ 超大 body 内存 DoS → readTextWithLimit 流式 4MB 上限 ⑧ fake-ip 伪造关 pin（攻击者首次 DNS 返 198.18 诱导关 pin 再 rebind）→ fake-ip 默认 fail-closed 拒绝 + `NOE_SSRF_ALLOW_FAKEIP` opt-in ⑨ pinnedLookup/特殊地址硬化。
- **我 51835 实测发现 codex 沙箱测不到的真实环境 P0**：codex 的"auto pin 闭合 TOCTOU"在 owner 中国+Clash 环境破坏抓取——Neo `server.js` 走 `EnvHttpProxyAgent` 代理，pin（无代理 Agent）覆盖代理直连，GFW 网站(wikipedia)抓不了（51835 实测 fetch failed）→ 修：`HAS_GLOBAL_PROXY` 有全局代理则不 pin（走代理），无代理 direct 才 pin。

**最终 51835 实测（新代码）**：wikipedia(GFW)✅走代理 5000字 + example.com✅ + `::127.0.0.1`/`127.0.0.1:51835`/`169.254.169.254`(元数据) 全🛡️拦。全量 **5321/5321 全绿**，ssrf-guard 单测 27 例（TOCTOU/redirect/fake-ip/pin-vs-proxy 全覆盖）。

**关键教训**：① 安全代码"源码存在≠真安全"——codex 六轮每轮都挖出更深的洞 ② "真实运行证据胜过理论"——codex 沙箱(无 GFW/无代理)测 pin 工作，owner 真实环境 pin 破坏 GFW 抓取，只有 51835 实测能暴露 ③ owner 环境(中国+Clash 代理)下 pin 与代理是根本矛盾，TOCTOU 防护靠"预检+逐跳+本机 owner-token+仅127.0.0.1监听"而非 pin。
