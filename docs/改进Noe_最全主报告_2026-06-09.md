### 3.2 方向岔路（最大选择题，不替 owner 拍板）

- **OS vs Agent 平台**：Noe 想做"完整 Jarvis 个人 OS"，但 2026 跑出来的是"轻 Agent + Skills + 跨端"（OpenClaw / Hermes / OpenJarvis）和"端侧+主动触发+MCP"（小跃/Qira/Marvis）。**所有"完整 Jarvis 个人 OS"明星硬件都死了**（Humane Pin 8.5 亿→1.16 亿被 HP 收购；Rabbit R1 4 天 5 万台→停薪+罢工）——不是"做得不够好"而是"用户刚需不存在"。owner 必须先答"做 OS 还是 Agent 平台"，否则后面 P0/P1/P2 都建在错地基上。
- **公开 vs 私有**：`package.json` `build.publish: github/owner:BB20260410/repo:noe` 与 README "私有项目未开源" 矛盾（N6 已验证）。
- **Electron vs Tauri**：Tauri 体积省 90%/内存省 75%/启动快 5-10x 是事实，但单人 dev 切 Tauri 需 2-4 周学 Rust+重写后端，**本轮不主张当前迁移**；中间态=保 Electron 优化资源占用，0 切换成本省 20-30% 内存。

### 3.3 P0-03 安全封装 = 发布阻断（为什么）

## 4. 记忆/梦境专题（Noe 该建的"睡眠记忆整合"）

> 竞品三家离线整合对比 + Noe ground-truth（grep 确认 salience/dreaming/unhide/recall-heat **全空**=真缺）。Noe **已有全部存储原语**：MemoryCore 已有 `hidden/hidden_reason/merged_into/merge_trace/confidence/hit_count/expires_at`，`merge()` 已会软隐藏 source。最该补的是"梦境调度 + salience + recall-heat + restore"这套结构件。

### 4.1 三家机制（去重后精华）

- **BaiLongma**=可上线的睡眠固化闭环：`consolidation-loop.js:34` 进程内 setInterval（启动 5min 首跑、之后每 30min），`consolidator.js` LLM(温度0/关 thinking/tool-call-only) 做①语义去重 merge ②旧并新 supersede ③陈旧降 salience ④**salience=5 身份级保护** ⑤**矛盾两条都留** ⑥无事 skip；drops 软隐藏(行+FTS+embedding 全留)。**自身缺口**=承诺可恢复但全仓无真正 restore。
- **OpenClaw**=教科书分阶段 dreaming（`extensions/memory-core/dreaming-phases.ts`）：Light(浅睡)收当日信号+去重+主题聚类 → REM(深睡)深度反思+提炼 **Candidate Truths**(带 confidence)；**recall-heat 晋升**：被≥3 个不同查询主题高频命中(uniqueQueries, MIN_SCORE=0.75/MIN_RECALL_COUNT=3)的短期记忆→rehydrate→写显式 `MEMORY.md`；**双轨**=向量/FTS 是潜意识、.md 是显式记忆（整合后真理写人类可读 .md 预加载省检索算力 + 主人可手改）；`memory-budget.ts` 严格 10K 字符预算按时间踢最老但保人工区。
- **Hermes**=会话后即时反思（非离线睡眠）：`background_review.py` 影子 agent 嗅探 + `curator.py` 技能 GC。思路可借，机制不如前两家成体系。

### 4.2 Noe 该建的设计（两层落地）

**A. 结构件（纯逻辑/无 LLM，可立即实现+单测）**
- `salience(1-5)` 列 + 打分锚点（5=身份级），给 NoePersonCards/身份 scope 设**硬保护白名单**（整合器永不可 merge/hide，比纯靠 LLM 判断稳）。
- **recall-heat 评分**：每次 FTS 命中入 prompt 记 `hit_count/last_hit_at/unique_query_count`，≥3 个不同查询主题命中→`promotionCandidate`。
- **软隐藏 + 一键 restore**：复用既有 hidden/merged_into，补 `unhide(memId)` 写回入口（顺手治 BaiLongma 自身缺口）。
- **注入期 rerankByImportance**：salience≥4 提前、>365 天陈旧下沉（stable sort，不改库）。
- **梦境调度骨架**：进程内 setInterval（可配间隔，**默认 OFF**，非 cron 合规），`NoeMemoryConsolidator` 纯函数仲裁层先跑**确定性规则**（精确重复/超 TTL/低 confidence），不调 LLM。

**B. LLM 整合器（需授权后台配额，碰 CLAUDE.md「默认禁后台 spawn 子 LLM」红线）**
- 移植 BaiLongma consolidator 提示词（去重/旧并新/降级/salience=5 保护/矛盾保留/skip，温度0+tool-call-only）+ OpenClaw Light/REM 分阶段 + Candidate-Truths 写入显式 `NOE_MEMORY_CORE.md`，走 Noe 多模型协作协议或默认关/手动触发，留 raw output 审计。
