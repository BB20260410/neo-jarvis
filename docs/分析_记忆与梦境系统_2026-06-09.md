## ① 三家"梦境/离线记忆整合"机制对比

### BaiLongma —— 整合循环 (consolidation loop)，可上线的"睡眠固化"
- **调度**：`src/memory/consolidation-loop.js:34` 进程内 setInterval，启动后 5min 首跑、之后每 30min 一 tick；`index.js:1533` 接线。后台空闲反刍，与主对话解耦。
- **选材**：`db.js:713 getCandidateEntitiesForConsolidation` 只挑 fact/person 且 visible、同实体记忆≥3 的实体，round-robin 轮流，`getMemoriesByEntity(e,20)` 喂全量上下文。
- **整合器**：`consolidator.js:4` LLM(温度0/关thinking/tool-call-only/三工具)：①语义去重 merge ②旧并新 supersede ③陈旧 downgrade 降 salience(不删) ④**salience=5 身份级保护** ⑤**矛盾两条都留(矛盾是信号)** ⑥无事 skip。
- **落地**：`executor.js:2191 execMergeMemories` keep PATCH、drops **软隐藏**(visibility=0/merged_into/hidden_at,行+FTS+embedding全留可恢复)；`execDowngradeMemory` 只降 salience。merge 标 high 权限(删数据)。
- **缺口(BaiLongma 自身)**：软隐藏号称可恢复,但全仓**无真正 restore(UPDATE visibility=1)**,只是注释承诺。

### OpenClaw —— dreaming，教科书级分阶段离线整合
- **真身**：`extensions/memory-core/src/dreaming-phases.ts` 是真离线引擎(UI 只是观测看板)。仿睡眠分阶段：
  - **Light Dreaming(浅睡, :1629)**：收当日会话信号 + 记忆提取日志,去重 dedupeEntries + 初级主题聚类。
  - **REM Dreaming(深睡, :1728)**：深度反思 buildRemReflections + 提炼 **Candidate Truths**(带 confidence, :1512),子 agent 分析短期记忆提炼高置信事实。
- **晋升机制**：`short-term-promotion.ts:2320 applyShortTermPromotions`,阈值 `MIN_SCORE=0.75 / MIN_RECALL_COUNT=3`,被**≥3 次不同查询主题高频命中**(uniqueQueries)的短期记忆→"补水重构 rehydrate"→写入显式 `MEMORY.md`。
- **双轨记忆**：LanceDB 向量(潜意识,自动捕获+L2 距离自动召回)+ Markdown FTS(显式固化)。**核心教训：向量/FTS=潜意识,.md=显式记忆**;整合后的真理写成人类可读 .md 预加载,省检索算力 + 主人可手改。

### Hermes —— 无显式"梦境",但有等价"后台审查 + 技能 GC"
- `agent/background_review.py`：会话后**异步影子 agent** 嗅探"用户修正/挫败/绕过技巧"→静默打补丁技能 + 隐式提炼偏好记忆(免 /remember)。
- `agent/curator.py:268 apply_automatic_transitions`：技能 GC——归档过期、合并重叠、吸收进全局准则(类比记忆整合,但作用于技能)。
- `conversation_compression.py / context_compressor.py`：近期高保真 + 早期摘要的无界上下文窗口。
- 判定：Hermes 是"在线/会话后即时反思",**不是离线睡眠期整合**;思路可借,机制不如前两家成体系。

## ② Noe 记忆现状与缺口(ground-truth)

Noe **已有**：`MemoryCore`(FTS5 trigram)、`NoeActiveMemory`(时间召回)、`FocusStack`(焦点栈带压缩)、`NoeKnowledgeGraph`、`NoePersonCards`(本轮新加)、`embeddings/`;且 MemoryCore 已有**软隐藏原语**:`hidden`/`hidden_reason`/`merged_into`/`merge_trace`/`confidence`/`hit_count`/`expires_at`,`merge()` 已会软隐藏 source(MemoryCore.js:200)。

Noe **缺**(真候选)：
1. **离线整合/梦境循环本体**：记忆层零调度器,记忆只在读写时被动变化,从不"睡眠期"主动清理。
2. **LLM 记忆整合器**：`merge()` 只是手动 API,无"自动发现语义重复并决定怎么合"。
3. **salience 显著性量纲**(1-5):只有 confidence/hit_count,无身份级保护(=5)/降级所需的统一信号。
4. **软降级/遗忘**:只有硬 TTL 过期,无"慢慢降重要性"与注入期"陈旧下沉/重要提前"重排。
5. **recall-heat 晋升**:无"被多查询高频命中→晋升显式记忆"机制。
6. **一键恢复 restore/unhide**:软隐藏可读(get includeHidden)但无写回复活入口。
7. **显式 .md 记忆核**:无 OpenClaw 式 `NOE_MEMORY_CORE.md` 预加载真理层。

## ③ 推荐 Noe「睡眠记忆整合」设计(融合三家 + 复用 Noe 原语)

分两层落地,**结构件可立即做,LLM 整合器需你授权**(碰 CLAUDE.md「默认禁后台 spawn 子 LLM 烧配额」红线):

**A. 结构件(纯逻辑/无 LLM,可现在实现+单测)**
- `salience(1-5)` 列 + 写入打分锚点(5=身份级);给 NoePersonCards/身份 scope 设**硬保护白名单**(整合器永不可 merge/hide,比纯靠 LLM 判断稳)。
- **recall-heat 评分**:每次 FTS 命中入 prompt 记 `hit_count/last_hit_at/unique_query_count`;≥3 个不同查询主题命中→`promotionCandidate`。
- **软隐藏 + 一键 restore**:复用 MemoryCore 既有 hidden/merged_into,补 `unhide(memId)` 写回入口(治 BaiLongma 自身缺口)。
- **注入期 rerankByImportance**:salience≥4 提前、>365 天陈旧下沉(stable sort,不改库)。
- **梦境调度骨架**:进程内 setInterval(可配间隔,默认 OFF),`NoeMemoryConsolidator` 纯函数仲裁层(给定一组记忆→产出 merge/downgrade 计划),先用**确定性规则**(精确重复/超 TTL/低 confidence)跑,不调 LLM。

**B. LLM 整合器(需你授权后台配额)**
- 移植 BaiLongma consolidator 提示词(去重/旧并新/降级/salience=5保护/矛盾保留/skip,温度0+tool-call-only)+ OpenClaw Light/REM 分阶段 + Candidate-Truths 写入显式 `NOE_MEMORY_CORE.md`。
- 走 Noe 多模型协作协议或默认关/手动触发,留 raw output 审计。
