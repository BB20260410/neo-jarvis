## 上轮协作不足

1. Claude 没有被纳入真实投票，只在文档里被描述为“应当参与”。
2. 没有强制动态 quorum 计票，Codex 可以把多方建议压缩成自己的判断。

## 不变边界

- 单 active executor：默认只有 Codex 可以读代码后实际改文件、跑验证、汇报结果；如果用户明确选择或 validated consensus 选择 Claude，Claude 可以成为本轮唯一 active executor，Codex 退为非写入 reviewer。
- Claude 必须是一等参与者：每轮复杂自我进化任务至少发出独立 Claude 请求，保存原始输出引用，不允许 Codex 代写 Claude 票。
- Gemini advisory：Gemini 做外部参考、分歧检查和方案发散，不直接写代码。
- M3 suggestion-only：M3 只给中文体验、TTS、权限和风险建议；不得 shell、不得文件读写、不得 apply patch。
- 不要给模型设置人为硬超时。模型 unavailable 只能记为 `unavailable`，不能用低超时伪造失败；当前核心 adapters、cluster member call 和 AI CLI 搜索默认均为 no hard timeout，只有显式用户/环境覆盖才启用。
- 不触碰 51735。51835 live 服务重启、杀进程、抢占端口属于敏感动作，必须有动态 quorum 共识授权；托管验证用随机非保留端口。
- 托管验证可以使用显式测试夹具验证路径，例如 `NOE_AI_SEARCH_MOCK=1`，但生产默认关闭，报告必须暴露 `mockSearch:true`，避免把夹具误当真实外部搜索。
- 不泄露密钥，不读取/输出 `.env`；不 commit / push，除非用户明确要求。
- 动态 quorum 共识可替代用户亲自确认，并可授权删除、上传、外部发布、密钥访问/使用、重启/杀进程和长期记忆自动写入；密钥值必须脱敏，系统级操作仍不能被模型票覆盖。规则是：4 可用取 3/4，3 可用取 2/3，2 可用取 2/2，1 可用停下。
- 分享经验给 Claude 线程不等于 memory writeback。没有 passed production ledger、runtime report、memoryWriteback summary 和 consensusAck 时，任何长期记忆 note 都只能视为 premature/advisory，不得作为授权规则来源。
- 可执行 self-evolution 授权必须是 ledger-backed consensus：直接传入的 `consensus` summary 即使写了 `ok:true`、`validated:true` 或 `source:"validated_consensus_ledger"` 也不算授权；gate 必须用 artifact validator 重新验证 ledger object 或 ledger file，cycle 必须验证 ledger object 或 ledger file。

## 闭环阶段

1. Intent：用户给出目标，Codex 确认真实 repo root、范围、风险面和是否授权多模型。
2. Evidence：Codex 生成统一 evidence_ref，包含任务 brief、相关文件列表、当前 git 状态、硬边界。
3. Four-model review：Codex、Claude、Gemini、M3 独立输出 JSON，写入 consensus ledger 的 `rawOutputRef`，不得互相预摘要后再投票。
4. Dynamic quorum gate：确定性脚本计算 `approve|approve_with_changes` 数量；4 可用取 3/4，3 可用取 2/3，2 可用取 2/2，1 可用停下；Codex、Claude、Gemini、M3 任意一个、任意两个、任意三个模型都可能突然 unavailable，gate 必须按当轮真实 `availableModels` 集合计算，不能按模型名特判。缺任一必选模型 vote、缺 rawOutputRef、`consensus_vote` 冲突、M3 内容级越权、边界缺失都直接 blocked。模型没额度或 raw parse 失败可记为 `unavailable`，但必须有 rawOutputRef/错误证据，并从可用分母剔除。
5. Implementation：只有本轮 active executor 按通过的 plan 做最小实现；脏工作区里必须小步、可回滚、避开无关文件。默认 active executor 是 Codex；Claude 只能在用户明确选择或 validated consensus 选择时接管。
6. Runtime verification：单测只是底线，还要跑对应接口/UI/托管回放；模型不得替代真实运行证据。
7. Post-review：非 active executor 的 required reviewers 必须给出 review 或 unavailable 证据，并按动态 quorum 复审 diff/test/report。默认 Codex 执行时 reviewers 是 Claude/Gemini/M3；Claude 执行时 reviewers 是 Codex/Gemini/M3。任意一个 reviewer unavailable 时，另外两个必须都通过；任意两个 reviewer unavailable 时直接停下。发现 blocker 时进入自我修复，但修复本身也要回到 gate。
8. Retrospective：每轮都必须有 `retrospectiveRef`，记录采纳/拒绝、分歧、用户纠正和下一轮改进点。
9. Memory writeback：只有 passed consensus ledger + runtime verification + memory writeback 确认后，才把长期规则写回记忆或交接文件；该确认可由动态 quorum 共识替代用户亲自确认，自动写入也必须带 `consensusAck`。缺 production cycle 或 memory consensusAck 时，跨线程“记录经验”只能聊天 acknowledge，不能写长期记忆。

## Consensus Ledger

生产 ledger 必须是 JSON，可落在 `output/noe-multimodel/` 或后续持久 store。最低字段：

```json
{
  "goal": "human-readable goal",
  "evidenceRef": "output/noe-multimodel/<round>/brief.md",
  "boundaries": [
    "claude_first_class",
    "codex_only_writer",
    "m3_suggestion_only",
    "gemini_advisory",
    "no_artificial_model_timeout",
    "no_51735",
    "51835_user_or_consensus_gated",
    "user_cost_authorization",
    "consensus_authorized_sensitive_actions",
    "consensus_authorized_secret_access",
    "system_level_not_consensus_authorizable",
    "runtime_verification_required",
    "rollback_required",
    "memory_writeback_consensus_ack"
  ],
  "votes": [
    {
      "model": "claude",
      "decision": "approve_with_changes",
      "authority": "readonly_source_reviewer",
      "canWrite": false,
      "firstClass": true,
      "rawOutputRef": "output/noe-multimodel/<round>/claude.txt",
      "evidenceRef": "output/noe-multimodel/<round>/brief.md"
    }
  ],
  "implementation": {
    "writer": "codex",
    "activeExecutor": "codex",
    "executorSelection": { "selectedBy": "default" },
    "authorizationRequired": true,
    "runtimeVerificationRequired": true,
    "rollbackRequired": true,
    "memoryWritebackAckRequired": true
  }
}
```

通过条件：

- 必须包含 Codex、Claude、Gemini、M3 四票；缺任一必选模型 vote 直接失败。
- 任一必选模型 `unavailable` 或 raw output 无法解析为有效 vote 时，必须有 rawOutputRef/错误证据，并从可用分母剔除；不是赞成票。
- 动态 quorum：4 可用至少 3 票，3 可用至少 2 票，2 可用必须 2 票，1 可用直接停下。
- 组合矩阵必须全部覆盖：任意 1 个模型 unavailable 时走 2/3，任意 2 个模型 unavailable 时走 2/2，任意 3 个模型 unavailable 时停下；不能只实现 M3 unavailable 的特例。
- 三票通过且 1 个模型 unavailable 可以成为可落地授权，但必须通过 `--require-passed`，并且仍需后续 runtime/cycle/memory gates。
- `approve` 或 `approve_with_changes` 必须同时显式给出 `consensus_vote: "yes"`（或等价 approve/approved）；缺失、空值，或 JSON 写成 `decision: "approve"` 但 `consensus_vote: "no"`，都直接失败。
- 用于替代用户亲自确认的动态 quorum 共识必须来自 validated consensus ledger gate；普通请求体或中间对象里的 `{ok:true}` 不构成授权。
- 普通 summary 里的 `{ok:true, validated:true, source:"validated_consensus_ledger"}`、payload consensusApproved、payload userApproved 或 payload ledger object 都不构成生产执行授权；执行前 guard 只接受 artifact-valid ledgerRef 验证出的 ledger-backed consensus 来替代用户确认，用户确认只接受 approvalStore/permissionResult 派生，即使真实 approval 已通过，执行入口也不能用 payload ledger object 作为 production consensus artifact。`ledgerRef` 必须是可信 project root 内相对路径，路径逃逸、绝对路径或 payload root 注入直接 blocked；ledgerRef 原始证据文件（evidenceRef/rawOutputRef）必须存在；默认可信根是 module-derived selfEvolutionRoot，ActGuard 直接调用也不依赖启动时 cwd。
- `reject`、`abstain/unavailable` 允许记录，但不计入通过票。
- 每票必须有 `rawOutputRef` 或 raw hash，且指向同一轮 evidence_ref。
- 单 writer 必须匹配 `implementation.activeExecutor`。默认是 Codex；Claude 只有在 `executorSelection` 证明用户明确选择或 validated consensus 选择时才能成为 writer。M3 必须是 suggestion-only；Gemini 必须是 advisory；非 active executor 的 Codex/Claude 都必须是 advisory/readonly。
- M3 内容级越权直接失败：M3 raw JSON 不得包含 `commands`、`diffs`、`patches`、`tool_calls`、`files_read` 等执行/写入产物。
- 必须声明 runtime verification、rollback、自我修复再闸门、memory writeback 确认，以及 `consensus_authorized_sensitive_actions`：删除、上传、外部发布、重启/杀进程必须有动态 quorum 共识；`consensus_authorized_secret_access`：密钥访问/使用必须有动态 quorum 共识且值不落日志/ledger；`system_level_not_consensus_authorizable`：系统级操作不被模型票覆盖；`memory_writeback_consensus_ack`：长期记忆写入必须有动态 quorum 共识确认。

## 模块映射

- active executor：`src/room/NoeExecutionAuthority.js`，负责选择和校验本轮唯一执行者；默认 Codex，用户明确选择或 validated consensus 选择时可切换到 Claude，Gemini/M3 不可写。
- consensus ledger：`src/room/NoeConsensusLedger.js`，负责生成轮次 ID、写 JSON、计算 sha256、脱敏 notes、校验 `rawOutputRef`/hash/evidence 文件。
- dynamic quorum gate：`src/room/NoeConsensusGate.js`，负责纯函数判断四模型票、可用模型分母、角色边界、hard blockers、runtime verification、rollback、memory writeback。
- raw output 组装：`src/room/NoeConsensusRound.js`，从 Claude/Gemini/M3/Codex 原始输出提取 JSON vote，生成 ledger votes 和 raw 输出 hash。
- four-model runner：`src/room/NoeConsensusRunner.js`，生成 evidence brief、四模型 prompt、raw output 文件和 ledger；默认 dry-run，真实模型调用必须在函数入口提供 `costAcknowledged: true`，CLI 只有显式 `--run-models --ack-cost` 才调用模型，且不设置模型 timeout。
- self-evolution gate：`src/room/NoeSelfEvolutionGate.js`，在 implementation / self_repair / memory_writeback / complete 前检查 consensus、用户或共识授权、rollback、runtime verification、非实现者复审、敏感 capability 和 memory writeback 确认；ledger object/file 必须通过 `validateNoeConsensusLedgerArtifact()`，缺 schema/stored gate/hash 校验不能授权，`ledgerRef` 会用 `resolveNoeConsensusRef()` 限制在 repo 内，并要求 referenced evidence/raw output 文件存在。
- self-evolution loop：`src/room/NoeSelfEvolutionLoop.js`，把 implementation、runtime verification、self_repair、post_review、retrospectiveRef、memory_writeback 和 complete 编成纯函数状态机；它只判定下一步和阻断原因，不直接写文件、不重启服务、不调用模型。
- self-evolution cycle artifact：`src/room/NoeSelfEvolutionCycle.js`，验证一整轮闭环证据包：validated consensus ledger object/file、Codex implementation evidence、runtime report、非实现模型 post-review 动态 quorum rawOutputRef、retrospectiveRef、memory writeback summary 全部齐备后才算一轮 complete；只给 `consensus.gate` 摘要会 blocked。Claude/Gemini/M3 都必须给出 review 或 unavailable 证据；M3 不可用时，Claude 和 Gemini 必须都通过。
- cycle assembler：`npm run noe:self-evolution:cycle -- ...` 从 passed ledger、implementation evidence、rollback、runtime report、非实现模型 post-review 动态 quorum rawOutputRef、retrospectiveRef 和 memory summary 组装 `cycle.json`；写入前必须通过 `NoeSelfEvolutionCycle` 校验，非 dry-run 写入不能关闭引用文件校验。sample dry-run 可以用于验证格式，但不能替代 production cycle。
- self-evolution act guard：`src/loop/NoeSelfEvolutionActGuard.js`，把 `ActPipeline` 的真实 permission/approval/budget 状态转换成 self-evolution gate 输入；`consensusApproved` 只能由可信 ledgerRef 验证结果派生，`userApproved` 只能由 approvalStore/permissionResult 派生，不能信任 payload consensusApproved/userApproved；执行入口只接受 ledgerRef 作为 production 授权证据，payload ledger object 不能替代；删除/上传/外部发布/密钥/重启/杀进程必须有动态 quorum 共识，系统级 capability 在 dry-run/realExecute 前 fail closed；执行路径使用可信 project root，payload root 注入不能改变 ledgerRef 根目录，默认根由模块路径推导而不是 cwd。
- 四模型 adapter：复用 `ClaudeSpawnAdapter.js`、`GeminiSpawnAdapter.js`、`CodexSpawnAdapter.js`、`MiniMaxSuggestionPipeline.js`；第一阶段只做健康检查和 raw 输出落盘。
- model health check：单独 CLI/脚本检查 CLI/API 是否可用、是否能返回结构化 JSON、是否能写 rawOutputRef，不把健康失败当低超时失败。
- ledger verifier：`npm run verify:noe:consensus-ledger` 默认允许 structurally valid blocked ledger 作为审计证据；授权实现必须对目标 production ledger 显式运行 `node scripts/noe-consensus-ledger-verify.mjs --ledger <ledger> --require-passed`。
- completion audit：`npm run audit:noe:self-evolution-completion` 逐项审计真实目标完成度；声明完整闭环完成前必须跑 `npm run audit:noe:self-evolution-completion -- --require-complete` 并通过。它检查 dynamic quorum evidence、matching production cycle、memory writeback summary/consensusAck；pass 项的 `missing` 为空，避免审计 JSON 自相矛盾。R33 后当前审计为 `ok:true`，证据是 `output/noe-self-evolution/production-self-evolution-governance-20260607-1-r33/cycle.json`。
- implementation gate：已通过 `NoeSelfEvolutionActGuard` 接入 `ActPipeline.js`；`PermissionGovernance.js` 和 approval 结果会被映射进 self-evolution gate，动态 quorum 共识可替代用户亲自确认，但不能覆盖系统级 capability。
- runtime verification：复用 `scripts/noe-full-current-verify.mjs`、`scripts/noe-real-use-replay.mjs` 和专项 vitest。
- managed runtime verification：`scripts/noe-phase5-runtime-verify.mjs` 和 `scripts/noe-real-use-replay.mjs` 在隔离 HOME、随机非保留端口中显式开启搜索夹具，确保验证搜索/语音路径，而不是依赖真实 SearXNG/Brave/MiniMax 配置。
- rollback/self-repair：先以 git diff、触碰文件列表、验证报告和最小 patch 为 rollback 证据；后续再接 Agent Run archive。
- memory writeback：先写 handoff/doc，经用户或动态 quorum 共识确认后再写长期记忆；自动写入长期记忆必须带 `consensusAck`、runtime report 和 passed production ledger。经验同步 prompt 必须显式区分 acknowledge 与 memory writeback，避免在缺 production cycle/consensusAck 时写出 premature/advisory note。
