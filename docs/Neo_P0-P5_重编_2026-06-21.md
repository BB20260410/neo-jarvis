- **交付物**：DPO 数据质量门 + LoRA 训练脚本 + 训练前后保留测试集对比。

- **完成判定**：训练产 LoRA、保留测试集不掉点；质量门拦掉重复/单源/超长样本。

### P5-2 人格一致性回归
- **目标**：权重更新不漂走 Neo 人格。
- **执行逻辑**：建 owner 选定的"人格一致性"题库；每次 LoRA 后跑，掉点超阈值自动 revert。
- **交付物**：人格题库 + 一致性评测 + 自动 revert。
- **依赖前置**：P5-1。
- **完成判定**：故意训坏样本 → 一致性掉点被检出 → 自动 revert LoRA。

### P5-3 权重版本化与回滚
- **目标**：权重比代码难回滚，必须版本化可逆。
- **执行逻辑**：每个 LoRA 版本入仓（版本号 + 训练数据指纹 + 评测分），一键 revert 到任意历史版本。
- **交付物**：LoRA 版本库 + revert 工具。
- **依赖前置**：P5-1、P5-2。
- **完成判定**：可回滚到任意历史 LoRA 版本、评测分可追溯。
