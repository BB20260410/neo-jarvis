# Noe panel launchd 守护 · 安装说明

> 波次5 P2 产物（2026-06-10）。**按约定不自动安装**——以下命令由你（owner）确认后手动执行。
> 效果：登录自启 Noe panel（51835）+ 进程崩溃 15 秒内自动拉起，全天候可达（配合 Telegram 入站可离机召回）。

## 安装（两条命令）

```bash
cp "$HOME/Desktop/Neo 贾维斯/docs/launchd/com.noe.panel.plist" ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.noe.panel.plist
```

装完 panel 会立即启动（RunAtLoad）。日志看 `/tmp/noe-panel.launchd.log`。

## 想常开哪些能力

编辑 plist 的 `EnvironmentVariables` 字典，加键值后重新 load：

| 键 | 值 | 效果 |
|---|---|---|
| `NOE_DREAM` | `1` | 梦境记忆整合循环 |
| `NOE_MEMORY_GC` | `1`（或 `dry` 只看不删） | 记忆库定时 GC |
| `NOE_GEO_WEATHER` | `1` | 天气进预取池（问天气秒答） |
| `NOE_MEMORY_EMBED` | `ollama` | 语义双路召回（需本地 ollama + 嵌入模型） |
| `TELEGRAM_BOT_TOKEN` | bot token | Telegram 随时召回 Noe |
| `TELEGRAM_ALLOW_FROM` | 你的 user id | 入站白名单（强烈建议配） |

## 卸载 / 重启服务

```bash
launchctl unload ~/Library/LaunchAgents/com.noe.panel.plist   # 停 + 取消自启
launchctl kickstart -k gui/$(id -u)/com.noe.panel              # 强制重启（改完代码后用）
```

## 可选：长期任务 follow-up 定时器

`com.noe.long-task-followup.plist` 不是 panel 服务，也不常驻。它每 15 分钟跑一次：

```bash
cd "$HOME/Desktop/Neo 贾维斯"
npm run noe:long-task-followup
```

作用是刷新/落盘长期任务续跑状态：未到 observation 检查时间就写 `waiting`；到期且 report stale 时，自动刷新 `verify:noe:observation-status` 与 `verify:noe:continuous-autonomy`。它不会绕过 expectation、soak、Hermes audit 门槛。

每次运行还会同步刷新 `output/noe-work-map/latest.json`，让面板/下个窗口可以从 Work Map 直接看到 follow-up 状态、下一条命令和下次检查时间。

安装：

```bash
cp "$HOME/Desktop/Neo 贾维斯/docs/launchd/com.noe.long-task-followup.plist" ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.noe.long-task-followup.plist
```

卸载：

```bash
launchctl unload ~/Library/LaunchAgents/com.noe.long-task-followup.plist
```

日志：

```bash
tail -100 /tmp/noe-long-task-followup.launchd.log
tail -100 /tmp/noe-long-task-followup.launchd.err.log
```

## 注意

- 装了守护后，**手动 `npm start` 会撞端口**——二选一。
- 你 2026-05-29 做过开机自启精简（只留 Clash/Codex/终端 claude），装这个会新增 1 项登录自启——确认要再装。
- plist 用 `zsh -lc` 加载登录环境，nvm 的 node 与代理变量都能拿到，无需硬编码路径。
