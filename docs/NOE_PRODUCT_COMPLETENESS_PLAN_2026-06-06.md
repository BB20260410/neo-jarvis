# Noe 产品完善度计划

更新时间：2026-06-06

本计划删除商业化、公开发布、营销叙事、外部用户招募等目标，只服务一个目标：让 Noe 成为 owner 每天愿意打开、能稳定帮忙、不会增加负担的本地个人 AI 助手。

## 1. 总原则

- 产品可用优先于功能数量。
- 真实验证优先于配置看起来完整。
- 本地优先但不把“本地”当作安全证明。
- 模型工作时不设置硬超时；界面要清楚显示“正在听 / 正在看 / 正在搜索 / 正在思考”。
- 危险操作必须可预览、可确认、可撤销。
- 不做公开分发，不做商业叙事，不做增长功能。

## 2. 不做清单

| 不做 | 原因 |
|---|---|
| GitHub 发布 / DMG 分发 / 自动更新 | 现在会引入签名、公证、隐私、支持成本，和个人使用完善无关 |
| 商业化定位和对外卖点 | 会把注意力从真实体验拉到文案 |
| Tool marketplace | 攻击面和维护成本过大 |
| Tauri 重写 | 会消耗 2-4 周，不能直接改善当前语音/视觉/身份体验 |
| 外部用户招募 | 当前仍在快速修本机体验，外部反馈噪声太大 |
| 社交账号发布能力 | 外发风险大，先不做 |
| 大型 GraphRAG / PPR 重构 | 当前 Noe 的短板不是检索架构，而是日常交互稳定性 |

## 3. P0：稳定当前日常主路径

目标：打开认知页后，语音、视觉、身份、模型选择都不互相打架。

| 任务 | 具体动作 | 验收 |
|---|---|---|
| P0-1 M3 永久 suggestion-only | 移除旧 executor env 入口，测试 env 打开也无效 | `test:m3:suggestions` 通过；`src/` 不再引用旧 env |
| P0-2 语音播放稳定 | 保留 TTS 播放保护、可调打断阈值、推荐值 | 开外放/录屏时不再只播第一个字 |
| P0-3 视觉和人物识别分区 | 视觉描述写“此刻看到”，人物库写“人物识别” | 两个区域不覆盖，下面空间可继续扩展 |
| P0-4 模型选择真实可用 | 只显示可连接本地/线上模型，不可用要标红或禁选 | 切到 MiniMax 不回退本地；LM Studio 模型列表来自真实 `/v1/models` |
| P0-5 附件入口稳定 | 图片/文件拖放、添加附件、发送后状态提示 | 图片和文本附件都能被读到并进入对话上下文 |
| P0-6 状态提示统一 | 搜索、研究、语音、视觉、人脸识别都有工作中提示 | 用户能判断系统是忙、失败、还是待命 |

优先验证：

```bash
npm run test:p0:unit
npm run verify:noe:cognitive-runtime -- --base-url http://127.0.0.1:51835
```

## 4. P1：身份系统做成真正可用

目标：Noe 能识别“主人 / 已知人物 / 未知人物”，并把身份信息安全地注入对话。

| 任务 | 具体动作 | 验收 |
|---|---|---|
| P1-1 人物库表格完善 | 展示所有人物、关系、标签、人脸样本数、声纹样本数、主人绑定状态 | 能从列表点击进入每个人并修改 |
| P1-2 人脸多样本无上限 | 允许一张起用，多角度越多越稳，不设置硬上限 | 一张正脸能建档，多张样本能提升匹配稳定性 |
| P1-3 声纹采样反馈 | 录音太短/太静要给中文原因和重录建议 | 用户知道为什么失败 |
| P1-4 阈值可调 | 人脸、声纹、打断音量都能在 UI 调 | 调整后刷新不丢 |
| P1-5 识别证据显示 | 显示分数、阈值、匹配的人、失败原因 | 不再只显示“未通过” |
| P1-6 本地模板管理 | 可删除某个人的单个样本、全部样本、整个人物 | 不保存原图/原录音，只保存模板 |

## 5. P2：语音体验完善

目标：Noe 像一个可自然打断的桌面语音助手，而不是按钮录音 demo。

| 任务 | 具体动作 | 验收 |
|---|---|---|
| P2-1 VAD 校准面板 | 显示当前麦克风电平、起说线、打断线 | 用户能看懂为什么没录到或误打断 |
| P2-2 TTS 播放队列 | 新回复来时明确停止旧回复，避免音频重叠 | 长回复不截断，连续回复不混音 |
| P2-3 只响应主人模式 | 唤醒词/口令 + owner token + 声纹/人脸状态融合 | 非主人语音不进 LLM，只显示状态 |
| P2-4 失败恢复 | STT/TTS/模型失败时给短中文原因和下一步 | 不出现沉默失败 |
| P2-5 语音设置持久化 | 音色、音量、打断阈值、是否自动播放可保存 | 刷新和重启后不丢 |

## 6. P3：视觉和屏幕理解完善

目标：Noe 不乱猜，不把旧上下文当画面事实。

| 任务 | 具体动作 | 验收 |
|---|---|---|
| P3-1 最新画面优先 | 视觉回答只从最新帧/截图得出结论 | 不再出现“叔叔聊天”等旧上下文污染 |
| P3-2 视觉证据卡 | 显示来源：屏幕、摄像头、附件、时间 | 用户知道 Noe 根据什么回答 |
| P3-3 摄像头状态摘要 | 连续摘要“人在/离开/不确定/多人” | 主动陪伴只在可信状态下触发 |
| P3-4 视频附件抽帧 | 视频按关键帧摘要，不上传原视频 | 能回答“这个视频里发生了什么” |
| P3-5 视觉关闭明确 | 关闭视觉后任何视觉问题都明确说看不到 | 不假装能看 |

## 7. P4：记忆和上下文完善

目标：Noe 记住有用事实，少记闲聊噪声，并能解释记忆来源。

| 任务 | 具体动作 | 验收 |
|---|---|---|
| P4-1 记忆分层 | 偏好、人物、项目、短期上下文分开 | 不同 profile 使用不同记忆权重 |
| P4-2 记忆来源可见 | 每条记忆带来源、时间、置信度 | 用户能删错记忆 |
| P4-3 垃圾记忆过滤 | 闲聊、重复寒暄、模型幻觉默认不长期记 | 记忆列表可读、不过载 |
| P4-4 对话证据注入 | 回答涉及人物/视觉/附件时带内部证据摘要 | 少答非所问 |
| P4-5 本地 Wiki 收敛 | Wiki 只保留能解决问题的知识页 | 不再为了“知识库好看”堆文档 |

## 8. P5：安全执行完善

目标：Noe 可以帮忙做事，但不会擅自破坏文件或花费配额。

| 任务 | 具体动作 | 验收 |
|---|---|---|
| P5-1 执行预览 | 所有写文件/移动/删除前显示计划 | 用户确认前不执行 |
| P5-2 撤销窗口 | 文件整理/移动保留可撤销记录 | 误操作能回滚 |
| P5-3 权限分级 | read-only、write_text、safe_exec 分层 | UI 明确显示当前权限 |
| P5-4 派活真实状态 | 待启动、运行中、失败、完成都可见 | 不再“点了没反应” |
| P5-5 本地日志面板 | 错误、模型、路由、耗时、失败原因可查 | 排障不靠猜 |

## 9. 每周验收节奏

每周只收 1-2 个主题，不同时展开太多：

1. 周一：选本周主题和验收标准。
2. 周中：实现 + 自测 + 浏览器实测。
3. 周末：按真实使用回放 10 个任务，记录失败。

每周输出一个小报告：

```text
本周修了什么
真实体验改善在哪里
还会失败的场景
下周只做哪 1-2 件事
```

## 10. 下一步推荐顺序

1. 完成 M3 suggestion-only 锁死和接手文档。
2. 做人物库/声纹/人脸识别 UI 的证据显示和样本管理。
3. 做语音 VAD 校准面板，解决“太敏感 / 听不到 / 误打断”。
4. 做视觉证据卡，彻底避免视觉回答夹带旧上下文。
5. 做记忆来源和删除入口，避免长期记忆污染。
6. 最后才扩展执行能力；执行能力必须默认预览、确认、可撤销。
