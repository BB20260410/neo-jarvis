## 0. 辩论裁决 + 诊断结论（v1→v2 的依据）

### 0.1 双代理辩论：v1 被 4 道独立闸击穿（「修一道仍等于没修」）
| 闸 | v1 缺陷 | 谁揭示 |
|---|---|---|
| 致命 R1 | embedding 只装 needle weak（`0<hits<minHits`），但根因是 **hits=0** 事件（被 `:1211 continue` 踢出）；短 claim minHits=1 时 weak 是空集 | 红队 |
| 致命 R2 | fail-open 假设 embed 抛错，但 ollama 单条失败**不抛错**退 128 维 `fallback:true`，`cosineSim` 在 `Math.min=128` 下**静默算假相似度** | 红队/代码实证 |
| 致命 P1-003 | `avgSemanticCoverage` 用词面口径重算 → embed-only 事件词面=0 **反而拉低指标**（即便关联存在也测不出） | 代码实证 |
| 致命 P1-004 | embed 证据因 `selectClaimLinkedEvidenceMatches` 排序基于词面 → 排末位被 `maxLines=8` 截掉，**judge 看不到** | 代码实证 |

### 0.2 真实数据诊断（panel.db 344 条 expectation）
- **闸5（`p≥0.75` 高自信门）不是主因**：303/344=88% 预测 `p≥0.75`（avg_p=0.817），自信足够产 surprise≥2bit。
- **真主因 = judge 判证链**：90% 未判证（310/344）+ 判证的 34 条**全 APPLIED** + **`failed=0`（从不判 FAILED）**；结合 `decisiveUnknownRate=0.976` → judge 几乎全回 UNKNOWN（证据不足）。

### 0.3 R6 离线实验（真实 claim × 后续事件 qwen3-embedding 相似度）
- **4/6 claim 有强区分**（simMax 0.8 / 跨度 0.5，#340 有 5 个 >0.6）→ **embedding 能召回 judge 词面漏掉的语义相关证据，P1-A 方向证实可行**。
- 阈值 0.6 偏高（漏 #343 的 0.575）→ 改 **0.5 + env 可调**。
- #339 类（simMax 0.42 无跨度）本就无对应现实证据、**不可判证** → P1-D 预测分层排除。

## P1-A · judge 接 embedding（重设计，修 4 道闸 + 工程缺陷）

新建 `src/cognition/NoeExpectationSemanticRecall.js`（<180 行，注入式，`// @ts-check`）+ 改 `buildEventsEvidence` async。改动清单：

1. **修 R1（作用于 hits=0 人群）**：embedding 召回**不放 weak 分支**，而是对 **`hits=0`（needle 完全未命中）的事件**算相似度——那才是 0.044 根因主体。在 `:1211 continue` 之外，对 `hits=0` 但 `embedSimilarity >= threshold` 的事件也纳入 matched（标 `embedSimilarity`、`embedMatched:true`）。
2. **修 R2（dim+fallback 守卫，复用 `VectorIndex:89` 思路）**：recall 内显式 `if (er.fallback || er.vector.length !== claimEmb.length) continue`，**绝不裸 cosineSim**。
3. **修 P1-003（coverage 口径）**：`buildEvidenceClaimAlignment` 对 `embedMatched` 事件的 coverage 取 `embedSimilarity`（非词面 0），让 `avgSemanticCoverage` 真能升。新增 `embedActionMaxCoverage` 字段进 alignment。
4. **修 P1-004（select 排序）**：`selectClaimLinkedEvidenceMatches` 的 route/`sortEvidenceMatches` 加 `embedSimilarity` 维度，让 embed 证据进 evidence 文本时间线（不被 `maxLines=8` 截掉）。
5. **修 P1-001（取 .vector）**：`const { vector, fallback } = await embed(text, {...})`。
6. **阈值（R6/R7）**：`threshold` 默认 **0.5**（R6 数据弱关联 0.575 也召回）+ `NOE_JUDGE_EMBED_THRESHOLD` env 可调。
7. **锁 model（P1-008）**：注入 recall 显式 `model:'qwen3-embedding:0.6b'`（1024 维，非默认 nomic 768）。
8. **Promise.all 并行（P1-006）**：events 批量并发 embed（非串行 await，避免 25 次串行延迟）。
9. **eventRef（P1-007）**：recall 与 `buildEventsEvidence` **共享同一 rows 对象引用**（禁 clone），用对象引用作 Map/WeakMap 键。
10. **fallback 可观测（R8，复用 VectorIndex dimHealth 思路）**：recall 降级时记 `degraded` 标志，judge resolve 结果带「本次走语义/退 needle」，落账可查。
- **env `NOE_JUDGE_EMBEDDING` 默认 OFF**：`buildEventsEvidence` **双形态**——`recall=null`（OFF）返回**同步闭包**（逐字回旧路径，零回归，53 resolver 测试基线）；`recall` 非空才返回 async 闭包（judgeOne `:1335`/tick `:1468` 均已 await，唯一装配点 `server.js:1998`）。

## P1 消融对照验收门（修 R4 + 隐藏 p 门 + 可证伪）
