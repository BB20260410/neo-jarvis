## 1. Hermes 真正值得学的架构点

### 1.1 Agent loop 不是无限循环，而是可解释终止

核心文件：

- `agent/conversation_loop.py`
- `agent/iteration_budget.py`
- `agent/turn_finalizer.py`

关键机制：

- 每个 agent / subagent 有独立 iteration budget。
- budget 耗尽时不是静默结束，而是剥离工具后再请求一次总结。
- turn 结束时记录 `turn_exit_reason`、api_calls、budget、tool_turns、last_msg_role、response_len。
- 如果最后停在 tool result、空响应、截断片段，会给用户可见解释。

Neo 对应：

- `src/runtime/mission/NoeMissionRunner.js`
- `src/runtime/mission/NoeMissionReconciler.js`
- `src/runtime/mission/NoeMissionCriteriaEngine.js`

Neo 缺口：

- 已有 Mission Runtime、checkpoint、evidence gate，但终态解释还不够统一。
- `maxSlices` 达到时此前只返回 paused，不会持久写明“为什么停”。
- succeeded/blocked 没有统一 finalization artifact。

本轮已落地：

- 新增 `src/runtime/mission/NoeMissionFinalizer.js`
- 成功、阻塞、caller slice limit 都写 `mission_finalization` artifact。
- state 记录 `finalizationRef`、`completionExplanation`。
- 人为 slice limit 会把 mission 持久标成 `paused`，并写 `mission.paused_by_caller_limit`。

### 1.2 后台审查必须工具白名单化

核心文件：

- `agent/background_review.py`

关键机制：

- 每轮结束后 fork review agent。
- review agent 只允许 memory / skills 工具。
- 危险命令 approval callback 自动 deny，避免后台审查卡死或越权。
- review fork 继承主会话 runtime，但 skip external memory，避免污染真实外部记忆。
- 最后只向用户展示 compact action summary。

Neo 对应：

- `src/skills/AutoSkillExtractor.js`
- `src/skills/SkillExtractor.js`
- `src/skills/SkillCurator.js`
- `src/memory/FactExtractor.js`
- `src/runtime/mission/*`

Neo 缺口：

- 已有 skill extraction / curator，但缺一个统一的 “post-turn background review contract”。
- 后台学习如果走真实模型，需要明确工具白名单、写入 provenance、dry-run / apply 两阶段。

应融合：

- 新建 `NoeBackgroundReviewRunner`。
- 默认 dry-run，输出 candidate memory / candidate skill / candidate pitfall。
- apply 时只允许 memory candidate、disabled skill draft、review report 三类写入。
- 所有写入带 `write_origin=background_review`、`execution_context=post_turn_review`。

### 1.3 Memory provider 要异步但可 drain

核心文件：

- `agent/memory_manager.py`

关键机制：

- 内置 provider + 最多一个 external provider。
- 外部 provider 不可 shadow core tool。
- prefetch/sync 使用单 worker 后台队列。
- 慢/坏 provider 不拖住主响应。
- session end 时 bounded drain，避免无限等待。
- `<memory-context>` 有 streaming scrubber，跨 chunk 防泄露。

Neo 对应：

- `src/memory/*`
- `src/context/NoeTurnContextEngine.js`
- `src/context/NoeContextBudgeter.js`
- `src/runtime/NoeContextScrubber.js`

Neo 缺口：

- 已有脱敏和预算裁剪，但 external memory / LanceDB / wiki / graph 等 provider 未来接入时，需要统一的异步同步和 scrubber 规则。

应融合：

- 为长期记忆加 `NoeMemoryProviderManager`。
- 每轮 prefetch 和 sync 分离，sync 不阻塞主回答。
- provider 失败只进入 warning/evidence，不让主脑卡死。
- streaming memory-context scrubber 复用 Hermes 思路，但实现为 JS 状态机。

### 1.4 Skill curator 是“可恢复归档”，不是自动删除

核心文件：

- `agent/curator.py`

关键机制：

- 自动 transition 是纯逻辑：active / stale / archived。
- LLM pass 单独做 consolidation。
- dry-run 不 mutate。
- snapshot 先行。
- archive 可恢复，报告明确 consolidated / pruned / state transitions。

Neo 对应：

- `src/skills/SkillCurator.js`
- `scripts/noe-skill-curator.mjs`
- `tests/unit/skill-curator.test.js`

Neo 缺口：

- 当前 curator 已能分类 stale/archive candidate，但还没有 snapshot、archive restore、consolidated/pruned 机器报告。

应融合：

- `runSkillCurator` 增加 `snapshotRef`、`recoveryInstructions`。
- archive 永远只是 proposal，除非 owner 或 review gate apply。
- 增加 overlap/consolidation report，不直接移动文件。

### 1.5 Context compression 关键是“历史只当参考”

核心文件：

- `agent/context_compressor.py`
- `agent/conversation_compression.py`

关键机制：

- 压缩摘要前缀明确：旧任务不是当前指令，最新 user message 赢。
- 历史 heading 用 Historical，避免模型恢复旧任务。
- summarizer 不可用时 fallback deterministic handoff。
- multimodal token 估算考虑图片。
- 压缩前通知 memory provider。

Neo 对应：

- `src/context/NoeContextBudgeter.js`
- `src/autopilot/NoeTurnFinalizer.js`
- `src/context/ProjectContextBundle.js`

Neo 缺口：

- Noe 已有预算器和 handoff，但可以加强“旧摘要不是活动指令”的前缀和测试。

应融合：

- 给 Noe handoff / context summary 加 stale-task 防复活语义。
- 测试 latest user message 覆盖旧 handoff。

### 1.6 Self-evolution 不是自改成功，而是候选变体过门

核心文件：

- `hermes-agent-self-evolution/evolution/core/config.py`
- `evolution/core/constraints.py`
- `evolution/core/dataset_builder.py`
- `evolution/core/fitness.py`
- `evolution/skills/evolve_skill.py`

关键机制：

- 变体来自 synthetic / sessiondb / golden dataset。
- train / val / holdout 分离。
- candidate 必须过 size、growth、non-empty、structure。
- 可选全测试。
- holdout 不提升就不采用。
- 默认输出 artifact，不直接改主仓。

Neo 对应：

- `src/cognition/*`
- `src/skills/*`
- `scripts/noe-self-evolution-*`
- `tests/unit/noe-self-evolution-*`

Neo 缺口：

- Noe 自我进化治理更严，但 skill/prompt 变体评测还可以更像 Hermes self-evolution：固定 dataset、holdout、growth limit、adoption gate。

应融合：

- 新建 `NoeEvolutionCandidateGate`。
- 所有自我进化 patch / prompt / skill 先进入 candidate archive。
- 必须有 baseline / candidate / metric / holdout / rollback。
- 没有通过 gate 不写默认配置。

## 2. Neo 应该复制/融合/不复制

### 2.1 直接复制思想

- turn finalizer/explainer：已开始落地。
- background review whitelist：下一步 P0。
- curator dry-run + snapshot + recoverable archive。
- memory provider async sync + bounded drain。

### 2.2 融合而非照搬

- Hermes Python agent loop 不搬到 Neo。Neo 已有 Node runtime、Mission Runtime、ActPipeline、Freedom、review gate。
- Hermes gateway 多平台不立即照搬。Neo 当前先把 51835/live/mind panel 跑稳，再做入站。
- Hermes GEPA 不立即跑真实优化。先做 mock/fixture gate，再接真实云端预算。

- 后台 review 默认积极写技能：Neo 不能默认无审查写长期行为约束，必须先 candidate + review。
- 直接外部 memory provider 多开：Neo 只允许 provider manager 统一接入，避免冲突。
- 直接 self-evolution 写主仓：必须 PatchTransaction / EvidenceReconciler / tests / rollback。

### P4：Context Summary 防旧任务复活

验收：

- NoeTurnContextEngine 注入摘要不触发旧任务。
- Continuum handoff 新窗口只把旧内容当背景。

### P5：Self-evolution Candidate Gate

目标：自我改进不是“生成 patch 就采用”，而是候选变体过门。

任务：

1. 新增 candidate archive：baseline/candidate/dataset/metrics/rollback。
2. 增加 size/growth/structure/test gates。
