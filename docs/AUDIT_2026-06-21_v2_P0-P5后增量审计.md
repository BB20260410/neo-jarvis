## 4. P6+ 迭代方向(9 条 · 按优先级)

> P6+ 不应再走"更多自进化代码"路线,而应走"用户体验放大"+"跨设备"+"跨进程"+"借鉴池接入"。

### P6-1 · 完整 Jarvis UI(🔴 最高优先 · 4-8 周)

- **现状**:README 自承「完整 Jarvis 未完成」;当前 `public/app.js` 731 行是工程师控制台,非对话优先
- **方向**:
  - 对话优先 UI(类似 ChatGPT / Claude.ai)
  - 卡片式布局(可拖拽)
  - 移动响应式(PWA-ready)
  - 集成 GWT 可视化 + Affect 头像 + Focus Stack 焦点高亮
- **依据**:owner 自由立场下,完整 UI 直接放大 Neo 用户面
- **工作量**:4-8 周

### P6-2 · iOS / 跨设备(🟠 高 · 2-4 周)

- **现状**:当前无 PWA / 无 native app
- **方向**:
  - 阶段 1:PWA(Progressive Web App),用 Web Push + Service Worker
  - 阶段 2:远程 host(Neo 跑在 Mac 上,iOS 通过 HTTPS 远程访问)
  - 阶段 3:SwiftUI 包 PWA(WKWebView + native shell)
- **依据**:Voice Session + Vision 主动陪伴 + 内态 provider 都是为多端设计
- **工作量**:2-4 周(PWA)+ 4-6 周(native)

### P6-3 · 跨进程协作(🟠 中 · 8-12 周)

- **现状**:server.js 3108 行 monolith(从 3033 增 75 行),所有功能在一进程
- **方向**:拆为独立 Node 进程
  - `chat-server`(对话 + BrainRouter)
  - `memory-server`(MemoryCore + SqliteStore)
  - `watcher-server`(Claude/Codex/Ollama/MiniMax adapters)
  - `voice-server`(STT/TTS/VAD)
  - `vision-server`(截图/VLM)
  - `self-evolution-server`(自进化 + 学习)
  - IPC:Unix socket(本机) + WebSocket(跨机)
- **依据**:Neo 真自改闭环跑通后,单进程已成瓶颈;跨进程后才能分布式
- **工作量**:8-12 周

### P6-4 · 分布式记忆 / 知识图谱(🟡 中 · 4-6 周)

- **现状**:SqliteStore 1131 行撑不住长跑(5749 测试 + 几个月记忆)
- **方向**:
  - 抽象 `MemoryStore` interface
  - 加 `libsql`(SQLite fork,支持 HTTP / WebSocket)
  - 加 `Postgres adapter`(可选)
  - NoeKnowledgeGraphTemporal 双时态已就位,可直接接分布式
- **工作量**:4-6 周

### P6-5 · Neo 4 房 → Neo N 房(🟡 中 · 4-6 周)

- **现状**:CrossVerifyDispatcher 3328 行(最大文件),4 房写死
- **方向**:
  - 重构为 handoff protocol(借鉴 `openai/swarm` 极简 handoff)
  - dispatcher 改 Map 注册 plugin
  - 支持 N 房动态注册 / 卸载
  - 借鉴 `camel-ai/camel` inception prompting 角色扮演
- **工作量**:4-6 周

### P6-6 · 真人陪伴体验层(🟡 中 · 2-4 周)

- **现状**:Affect 引擎已有,VAD 已就位,但 UI 不可见
- **方向**:
  - 头像 + 情感色彩(红 = arousal 高 / 蓝 = valence 低)
  - TTS 情感色彩(语速 / 音调随 VAD 变)
  - 思维气泡(GWT 当前焦点可见)
  - 屏幕保护模式(空闲 30min → 进入柔和陪伴态)
- **工作量**:2-4 周

### P6-7 · 自演化循环提速(🟢 低 · 1 周)

- **现状**:owner 自由立场下,心跳 cooldown 可放开
- **方向**:
  - `cooldownMs` 30min → 5min 可配
  - 心跳拍 60s → 30s 可配
  - owner 完全控制
- **工作量**:1 周

### P6-8 · 端到端加密记忆(🟢 低 · 2-3 周)

- **现状**:记忆跨设备同步未加密(本机 OK,跨机后裸奔)
- **方向**:
  - 用 `libsodium` sealed box(E2EE)
  - owner 设备密钥 = 主密钥
  - 跨设备同步需 owner 显式授权 + 设备密钥对
- **工作量**:2-3 周

### P6-9 · 自我讲故事 / 反思日记(🟢 低 · 2-3 周)

- **现状**:NoeLearningLoop 已学,但 Neo 自己不讲故事
- **方向**:
  - 每周 `NoeNarrativeReflection` 心跳自动写周记
  - 写进 `noe-self-narrative` 记忆
  - 跨周可对比"上周我做了什么"
- **工作量**:2-3 周

- **未来 LoRA 训练 gate 必含**:
  1. 注入 Noe system prompt
  2. 关 thinking
  3. 抓 base 身份泄漏
  4. 抓英文推理前缀
  5. 正面人格要求
- **数据**:按"温暖陪伴 vs 项目复盘"分层,只用前者训

### 6.2 base 模型路径 + 离线兜底前置

- **现状**:Neo base 模型路径 `~/.lmstudio/models/lmstudio-community/Qwen3.6-35B-A3B-MLX-6bit`,必须 `NOE_LORA_BASE=该路径` 绕开 HF 下载
