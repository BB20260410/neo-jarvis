## 关键设计决策（踩坑/取舍，下窗口别重蹈）
1. **不上影子 worktree**（沿用 executors:183-190 既有结论）：Neo 工作树长期领先 origin+大量未提交+better-sqlite3 native binding → worktree 测旧代码无意义。P3「行为不变」改用 **双绿门**（apply 前 baseline 绿 + apply 后 verify 绿，`runtimeVerify({root})` 已支持裸跑全量 test）+ 既有 backup/原子 rollback 事务性。
2. **P3 gate 始终注入**（不 flag 门控注入），`NOE_EVOLUTION_LOGIC` 只控「是否放行改逻辑」：默认 OFF=改逻辑 patch 被 preCheck 早拒+rollback（最安全），恰好接住 P2 打开的 high_complexity 改逻辑口子。doc_only/neutral 飞轮零影响（preCheck 放行）。
3. **test_only 豁免是 dead path**：P0-5 安全门拦 `tests/` 写入（防 reward hack 改测试骗 verify）→ Neo 自补测试 patch 必被 preflight 拦。test_gap 信号价值=提示人类补测试，非 Neo 自补。gate 的 test_only 分支保留为前瞻（单测覆盖，无害）。
4. **P5 advisory-only 硬隔离**：`NoeMetaEvolution` 接口物理上无任何 mutate flag/gate/config 能力，只有 read(outcomeStats/flagSnapshot)+writeAdvisory。即便诊断「该开 logic」也只写文字建议给 owner。安全机制永远在 Neo 控制之外（owner 钦定）。
5. 防刷屏：P4 太浅教训 + P5 建议均 24h 同 title 去重；P4 游标存 `evolution_retrospect` 事件 payload.cursor（重启不重复复盘）。
