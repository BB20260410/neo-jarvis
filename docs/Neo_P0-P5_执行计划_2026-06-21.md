**P0-1 让 codex implementer 真出 patch_plan【✅ 已完成 2026-06-21】**
- **真因**:codex(gpt-5.5)天然产出 `op:replace`(from/to)增量 diff,而 NoePatchTransaction/NoePatchApplyExecutor 只认 `op:write_file`+整文件 content → `unsupported_patch_operation` 拒。schema 对 codex 无效(CodexSpawnAdapter._doChat 忽略 response_format,出啥格式靠 IMPLEMENTER_SYSTEM 自然语言引导+概率)。非解析问题(解析鲁棒早已实现)。
- **修复**:全链路支持 `replace` 增量 op(parsePatch/checkPreconditions/apply/summarizeOperation + IMPLEMENTER_SYSTEM/schema)。from 须文件内精确唯一,函数式 replace 避 `$&` 解释。
- **审计加固(3 轮对立子代理 + multimodel,逐轮 FAIL→迭代)**:① checkPreconditions **虚拟串行应用**(防同文件多 op 唯一性基于原盘漂移→错位/丢失)② finalText secret **值集合差集**(防拆分注入 + 净持平对冲,extractSecretLikeValues 比对原始 secret 值)③ NUL 清除(sha 连接符)④ rollback 接 PolicyFileGuard ⑤ from/to 长度 cap ⑥ apply 末道 occ 守门。

- **完成判定:✅ 已达成**(e2e + 真仓 dryRun + 全量绿 + 3 轮审计)。剩活体心跳自主调度——p0fix goal 因历史 churn(34 checkpoint)被 server 启动改回 `done`,致 openSelfEvolutionGoals 跳过、心跳不 tick;需 P0-5 理顺 goal 状态机/churn 自消(或新建 open goal),才能让心跳自动选到 ready goal 真改。能力已铁证,差此调度环。
