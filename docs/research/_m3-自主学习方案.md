# Neo 自主学习：从"原地打转"到"真知识增量"——可落地工程方案

## 0. 核心判断先说

根因**没遗漏**，是**学习循环缺一个"是否还在学"的判据**。补上**4 个度量信号 + 1 个动态选题器**，原有架构（surprise / expectation 账本 / 好奇回路 / noe_memory / 目标系统 / 深思）几乎不动。**改造面积 < 6 个文件，~400 LOC，全部可证伪。**

## 1. 业界怎么解决"重复探索同样东西"（带出处）

| 方案 | 关键机制 | 给 Neo 的启发 |
|---|---|---|
| **ICM**（Pathak 2017, ICCV best paper） | forward model 预测 φ 特征、预测误差 = intrinsic reward | Neo 已经有 **NoeExpectationLedger**（p→outcome→surprise），就是 ICM 的特例（用元数据当 φ） |
| **RND**（Burda 2018, ICLR'19） | 随机固定 target net，predictor 拟合误差 = novelty | Neo 蒸馏后 embedding 与历史 centroid 的**距离**就是 RND 信号 |
| **Schmidhuber compression progress**（1991→2010） | reward = 编码器长度 − 先验长度的**下降速率** | Neo 的 **skill_distill 长度** 突然缩短 = 学到压缩 = 真信号 |
| **Go-Explore**（Ecoffet 2019/2021） | **显式归档** cell 状态 + 从归档点重启 + 持续 novelty bonus | Neo 应该归档"已深度读过/蒸馏过的页面/主题 hash"，读前先查 |
| **Voyager**（Wang 2023） | 技能库自演化 + 课程自动生成（GPT-4 评 difficulty） + curriculum 渐进 | Neo 的 **noe_memory** + skill_distill 已经具备，要加"技能尝试成功率"反馈 |
| **Novelty Search**（Lehman 2011） | 只奖励 novelty、不奖励 objective；behavioral distance 进归档 | Neo 已经 surprise-driven 选目标，缺的是**归一化**和**饱和检测** |
| **Open-ended**（Sutton Silver 2023） | 永远最大化"可学习的量" → 元目标：环境产生新可学 | Neo 的 surprise 越大 = 该方向越可学，但需加"已榨干"检测 |
| **EWC / SI / Replay**（continual learning） | 显式防灾难性遗忘（Fisher 矩阵 / 路径积分 / 经验回放） | Neo 不训练 NN，但**经验回放**直接可用：定期从 noe_memory 抽旧 topic 重读 |

**业界共识**：单纯"surprise 驱动"会陷入**noisy TV trap**（永远奖励新观察、永远不收敛）。Burda 2018 已经用 RND 验证。所以 Neo 的 surprise 只能做**初筛**（该不该开这方向），**不能做终止信号**。终止要靠"知识饱和度"。

## 2. 业界怎么"动态决定下一个学什么"

**不要"上次的"或"固定的"，要用"上次学完后的 surprise 余量"**。三种主流做法：

### A. Schmidhuber 法：maximize learning progress（推荐 Neo 用）
- 给每个 topic 维护**两条 EMA**：`quality = how much does learning it reduce loss?`
- 选 `argmax (quality_ema + novelty) / cost`
- **质量下降 = 学完了**（边际知识增量 < ε）

### B. Go-Explore 法：归档驱动（推荐 Neo 加）
- 维护 `archive: Map<topicHash, {nVisit, lastSurpriseMean, lastSaturation}>`
- 优先访问**未访问**或**低访问 + 高余量**的 cell
- 杜绝"6 个写死"

### C. Voyager 法：GPT 评 difficulty → curriculum（Neo 可选）
- 学完一个技能后，让 LLM 提议"下一步该学 X 因为 Y"
- **风险**：LLM 提议 = 新一类写死。需要 surprise 余量反向校验

## 3. 4 个信号：区分"真知识增量" vs "重复蒸馏同质方法论"

这是空耗的**根诊断器**。每开一个目标前要算，蒸馏后要更新：

| 信号 | 定义 | 阈值（建议初值） | 含义 |
|---|---|---|---|
| **S1 = Embedding Novelty** | 新读内容 embedding 与该 topic 历史 centroid 的 1 − cos | < 0.15 = 重复 | 真读新东西 |
| **S2 = Compression Δ** | `len(新 skill_distill) − len(旧 skill_distill)`，用 RLE 或 LLM-token 长度 | ≥ −10% 算"压出新结构" | 真学到了压缩表示 |
| **S3 = Surprise 衰减率** | 该 topic 最近 3 次开目标的 `harvestSurprise` 的 EMA 斜率 | slope > −0.3 bit/次 = 仍在 surprise 平台期 | 还在产出 surprise |
| **S4 = Closed-Loop Hit** | 蒸馏出的方法论**真的被某个 future goal 当 source 用** | ≥ 1 次引用算"反哺" | 真进入了 Neo 自己的工具箱 |

**判定规则**（任一命中即"真学到了"，全不中即"空耗"）：
```
REAL_LEARNING = (S1 ↑) ∨ (S2 显著 ↓) ∨ (S3 仍高) ∨ (S4 ≥ 1)
EMPTY_LOOP   = S1 < 0.15 ∧ S2 ≈ 0 ∧ S3 衰减 ∧ S4 == 0  (≥ 2 轮)
```

**关键：S4 是反 reward hacking 的核心。** 你蒸馏再多方法论套话，如果从来没被 NoeGoalSystem 当 source 引用过，就是空话。

```
┌────────────────────────────────────────────────────────────┐
│ 1. 选下一个 topic（NoeTopicCurator.tick）                   │
│    inputs:  archive, lastClosedLoopSat, ownerInterestVec   │

│                                                            │
│ 2. 立目标（NoeGoalSystem.maybeSeedAutonomousLearning）     │
│    source=self_learning, payload={topic, expectedNovelty}  │
│    cooldown = min(60min, lastGoalGap)         ← 关键改    │
│                                                            │
│ 3. 读内容（freedom executor）                              │
│    pre-check: archive.get(topic).lastSaturation ≥ 0.85     │
│             → 直接收尾、不读                                │
│    pre-check: topicVisited(url) within 7d → 换 URL         │
│                                                            │
│ 4. 蒸馏（skill_distill）                                   │
│    compute S1 = 1 − cos(emb, topicCentroid)                │
│    compute S2 = lenDiff(newSkill, oldSkill)                │
│    若 S1 < 0.15 AND S2 ≈ 0 → 标 [STALE]，不写入 skill_distill│
│                                                            │
│ 5. 写进度（NoeLearningProgress.recordVisit）                │
│    archive[topic] = {                                      │
│      nVisit++,                                             │
│      emaSurprise: update EMA,                              │
│      emaQuality: update from S2,                           │
│      lastSaturation: clamp(1 − S1·S3),  // 综合           │
│      lastClosedLoopHit: 0  // 等 §4.3                      │
│    }                                                       │
│                                                            │
│ 6. 深思反思（NoeDeliberation）                              │
│    如果最近 3 个 self_learning 目标 saturation 全 ≥ 0.85  │
│    → owner 推送: "这 3 个方向已饱和，建议换 X/Y/Z"        │
└────────────────────────────────────────────────────────────┘
```

### 4.3 闭环验证（ClosedLoopHit）—— 防 reward hacking 核心

```
A 目标蒸馏出方法论 M
  ↓
未来 N 天内 any goal.source === M.id (被 NoeGoalSystem 当 source 引用)
  ↓
命中 → archive[topic].lastClosedLoopHit = now
```

**NoeGoalSystem 选 source 时**优先从 skill_distill 里**真正被引用过**的挑。**引用 0 次的 skill 自动降权**，3 天后归档。

### 4.4 判据表（每项必须可证伪 / 可观测）

| ID | 判据 | 怎么验 | 失败动作 |
|---|---|---|---|
| **J1** | 7 天内自创目标中**重复目标**比例 < 5% | grep NoeGoalSystem 落库 | curator.score 调高 novelty 权重 |
| **J2** | 任何 topic saturation ≥ 0.85 后**不再被开** | archive 查 | curator.score 调高 saturation 罚 |
| **J3** | 蒸馏出的 skill_distill **7 天内至少 1 次被 goal 引用** | 引用计数 | 0 引用 skill 自动归档 |
| **J4** | 连续 5 次自创目标，**至少 3 个 topic hash 不同** | archive | curator novelty 权重 |
| **J5** | owner 推送建议接受率 ≥ 50% | owner 反馈 | 选题器降权"被拒"topic |

## 5. 风险与反例（必看）

### R1：Curator 自己陷入"换皮重复" ⚠️ 高
- **症状**：curator 选的 topic hash 不同、但 embedding 距离 < 0.2（都在 llm-agent 圈打转）
- **对策**：S1 用**全局**最近 20 个 topic 的 centroid 算，不只本 topic
- **可证伪**：J4 不再满足时

### R2：Surprise 永不下平台 = noisy TV ⚠️ 高
- **症状**：web 内容 surface-level 永远能产生 surprise，但都是废话
- **对策**：surprise 信号**只能入新方向**，**不能当进步信号**；进步靠 S2（压缩率）
- **可证伪**：S3 仍高 + S2 ≈ 0 连续 3 次 → curator 强制冷却该方向

### R3：LLM-curator 被 reward hack ⚠️ 中
- **症状**：curator 用 GPT-4 提议 topic，GPT 学会"提议 owner 爱听的话"
- **对策**：curator 提议后必须经过 **archive.saturation 过滤** 才进候选
- **可证伪**：owner 接受率下降

### R4：ClosedLoopHit 永远 0（鸡生蛋）⚠️ 中
- **症状**：skill_distill 没被引用 → saturation 永远高 → 不学 → 还是没引用
- **对策**：**前 3 个目标强制注入 1 个历史 skill 作 source**，打破死锁
- **可证伪**：J3 长期 = 0 时

### R5：Tab 堆积 ⚠️ 低（不修也行但建议修）
- **对策**：freedom executor 开 tab 前调 `archive.get(url).lastOpen`，> 1d 旧关掉
- **可证伪**：browser tab 数 > 20 时告警

### R6：Cont. learning 灾难性"覆盖" ⚠️ 低
- **症状**：noe_memory 滚动覆盖旧 topic
- **对策**：skill_distill 加 `pinnedUntil`，被引用过的 pin 30 天

## 6. 落地顺序（防引入新空耗）

| Phase | 内容 | 周期 | 验收 |
|---|---|---|---|
| **P0** | 删 `topicCursor = goalCountBySource`，加 60min cooldown | 0.5d | J1 7 天内 < 5% |

| **P2** | `NoeTopicCurator` 替换写死列表 | 3d | J4 满足 |
| **P3** | ClosedLoopHit 闭环 + skill 降权 | 3d | J3 满足 |
