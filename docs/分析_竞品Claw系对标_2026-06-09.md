## ① 速览结论

这几个都是"AI agent / 网关 / 生成 CLI"类，与 Noe 同域但**侧重不同**：
- **OpenClaw = Noe 最大互补面**：它是成熟的**多渠道入站网关**（让助手从你常用的聊天 App 可达）+ 一批工程级健壮性机制；Noe 目前只有**出站**社交发布链 + webhooks，**入站可达性是真缺口**。OpenClaw 有几条小而精的机制 Noe 可直接复刻。
- **Hermes = 与 Noe 自我进化正面对标**：但 Noe **已有** 技能提取/curation 体系 + consensus 门控自我进化（更严谨可审计）；Hermes 的"技能用中改进 + 跨会话深化用户模型"是可借的**局部增强**，整体不如 Noe 严谨。
- **MiniMax CLI = 能力工具**：Noe 应**集成调用**（Noe 已有 MiniMax 语音 key，扩展图/视频/音乐生成），不是复刻架构。

## ② 复刻/融合候选（按性价比排序）

### D1. 自动回复「代际栅栏」防连击竞态　【we_lack · 极高 · 工作量 M】
- **对方做法**：`src/auto-reply/dispatch.ts:50-180` `ForegroundReplyFenceState`：以 session×channel 为键 + **generation 代际计数器**，并发自动回复时只有最新 generation 可见投递，旧代静默压制。
- **Noe 现状**：`NoeLoop.js` 有 abortController 能打断 tick，但**无 session×channel 代际锁**——用户连发多条时旧回复可能覆盖新回复。
- **价值**：任何聊天/主动回复机器人必踩的坑。**可直接复刻这段纯逻辑**（见 P3 已落地）。

### D2. 承诺/待办提取 + dueWindow 时间窗跟踪　【we_lack · 高 · M】
- **对方做法**：`src/commitments/store.ts:25-100` 把"明天提醒我买菜"提炼为结构化 `CommitmentRecord`（类别 event_check_in/open_loop、敏感度 routine/care、**dueWindow earliestMs/latestMs**），后台在时间窗内触发而非死板 cron。
- **Noe 现状**：只有零散 followup（CollaborationDispatcher/AgentRunStore），无结构化承诺跟踪。
- **价值**：让 AI 有"记忆和责任感"。融合进 `src/memory/` + proactiveTick 触发。

### D3. 多渠道入站统一网关　【we_lack · 高 · L】
- **对方做法**：`src/gateway/server-channels.ts:40-150` `ChannelRuntimeStore` + 快照隔离 + 退避重连，解耦长连接与业务，单渠道断线不影响主路由。
- **Noe 现状**：只有 `src/webhook/`（出站/被动），无 telegram/微信/discord 入站。
- **融合方式**：先 EventEmitter 抽象 + 单渠道（telegram）试点，后续加退避重启。大工程，建议立项分期。

### D4. 多模态生成能力集成（MiniMax）　【we_lack · 中 · S】
- **对方做法**：`mmx-cli` 文/图/视频/语音/音乐/vision 统一 CLI。
- **Noe 现状**：只用了 MiniMax 语音；无图/视频/音乐生成。
- **融合方式**：Noe 已有 MiniMax key，把图/音乐生成加为 freedom capability（调 MiniMax API 或 mmx-cli），非复刻架构。注意烧 MiniMax 额度。

### D5. 上下文引擎 Proxy 隔离降级　【we_worse · 中 · S】
- **对方做法**：`src/context-engine/registry.ts:60-100` 用 JS Proxy 包装上下文引擎，致命错误即打入 quarantine，后续降级走内存回退，保聊天不断联。
- **Noe 现状**：`NoeContextEngine` 故障隔离待确认；可加 Proxy quarantine。
- **价值**：几行 Proxy 即可的健壮性典范。

### D6. AsyncLocalStorage 诊断时间线　【we_lack · 中 · M】
- **对方做法**：`src/infra/diagnostics-timeline.ts:20-100` 用 `AsyncLocalStorage` 无感透传 span，记 durationMs/p99/parentSpanId 落 JSONL，画火焰图不污染 stdout。
- **Noe 现状**：缺结构化异步性能追踪（有 metrics/ 但非 span 级）。
- **价值**：本地 Node AI 助手极缺的性能观测。

### D7. 心跳 HEARTBEAT.md + notify 门控（增强 proactiveTick）　【enhance · 中 · S】
- **对方做法**：`src/auto-reply/heartbeat.ts:20-100` 检测工作区 `HEARTBEAT.md` 有有效任务才注入隐形心跳对话，模型用 `heartbeat_respond` 的 `notify=true/false` 决定是否真打扰用户。
- **Noe 现状**：**已有** `src/loop/proactiveTick.js`（主动开口/甜心小玲），可借鉴"工作区文件驱动 + notify 克制打断"。

### D8. 用户级 launchd 免 root 自启　【reference · 小】
- **对方做法**：`src/daemon/systemd.ts` 用 `~/.config/systemd/user/` + linger 免 root 常驻。
- **Noe 现状**：macOS=launchd，已有开机自启（见记忆 reference_startup_items_optimization）。参考其无 root 模式即可。

## ⑦ Hermes 自我改进循环深读补充（Gemini agentic, 文件:行）

Hermes 招牌=自我改进循环。Noe 已有技能体系，但以下是可借的**局部增强**：
- **伞形技能目录范式**（`agent/background_review.py:80`）【极高·M】：技能=`SKILL.md`+`references/`+`templates/`+`scripts/`，把"知识"变"立等可取的工具包"。→ 升级 Noe `SkillStore` 的技能结构。
- **预算耗尽前"死前总结"**（`agent/turn_finalizer.py:53`）【高·S】：loop 预算将尽时不硬掐，剥夺工具权限强制写"做完了什么/卡在哪"的交接总结。→ Noe `ActPipeline`/budget 可加这层降级拦截，大幅提升无人值守体验。
- **后台技能提取 + 隐式记忆提取**（`agent/background_review.py:327/20`）【高·中/低】：会话后影子 Agent 嗅探"用户修正/挫败/绕过技巧"静默打补丁技能 + 提炼隐式偏好记忆（免 /remember）。→ 增强 Noe `AutoSkillExtractor`。
- **Curator GC**（`agent/curator.py:1420`）【高·高】：周期性合并重叠/重命名含糊/归档过期技能防"技能腐败"。→ 增强 Noe `SkillCurator`。
- **技能快照指纹冷启**（`agent/prompt_builder.py:975`）【中·低】：技能描述+目录指纹存 JSON 快照，指纹未变直接注入，秒级冷启。
- **子 Agent 知识返收钩子**（`memory_manager.py:761` on_delegation）【中·中】：子 Agent 经验回灌父会话记忆池。
- 已≥项：流式 `<think>` 清洗 Noe 已有 `NoeContextScrubber`（cleanVisibleModelText/redactSensitiveText）→ ≥ Hermes think_scrubber。

**Hermes vs Noe 总判**：Noe 自我进化（consensus 门控+审计）整体 **>** Hermes；但 Hermes 的"伞形技能目录""死前总结""隐式记忆提取"是 Noe 可低成本吸收的精华。
