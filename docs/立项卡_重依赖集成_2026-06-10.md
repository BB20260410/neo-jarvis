> Top100 裁决中标"📋 立项"的项,每张卡=可直接开工的入口(目标/步骤/验收/成本)。
> 共同前提:装包/下模型动作较重,单卡单窗口做透,装前告知 owner 体积。

## 卡① sherpa-onnx 语音实时化（A 簇首选,一项顶三项）
- **目标**:替换"整段 POST whisper(3-7s 断顿)"为流式 STT+VAD+唤醒词一体,实现"说完即应"。
- **步骤**:`npm i sherpa-onnx-node`(原生 addon)→ 下流式 zipformer 中英模型(~200MB,github releases)→ 新建 `src/voice/SherpaSttClient.js`(实现 LocalSttClient 同款 transcribe 契约+新增流式接口)→ VoiceSession 可切换 → VAD 端点接 `VoiceVad.js` 既有钩子。
- **验收**:实测同句话延迟 对比 whisper(目标 <1s 出首字);唤醒词"嘿 Noe"误触率人工测 10 次。
- **成本**:包+模型 ~300MB;一个专注窗口。

## 卡② CosyVoice 本地中文 TTS 兜底（B 簇首选）
- **目标**:断网/MiniMax 不可用时中文不哑(当前 Kokoro 仅英文)。
- **步骤**:独立 Python venv 装 CosyVoice(阿里官方 repo)→ 下 300M 流式模型(~2GB)→ 起本地 HTTP 服务(官方自带 fastapi 示例)→ 新建 `CosyVoiceTtsClient.js`(synthesize 契约同 MiniMaxTtsClient)→ `_pickTts` 链尾兜底。
- **验收**:断网拔 MiniMax 实测中文出声;音质 owner 实听拍板(口碑不算数)。
- **成本**:~2.5GB 磁盘;GPU 不必须(M5 Max CPU/ANE 可)。

## 卡③ RapidOCR 屏幕读字（C 簇轻量先手）
- **目标**:视觉链路能读屏幕具体文字(当前 qwen-vl 只懂语义不精确读字)。
- **步骤**:`pip install rapidocr-onnxruntime`(轻,~60MB 含模型)→ `scripts/noe-ocr.py` 包一层 stdin/stdout → `src/vision/OcrClient.js` spawn 调用 → VisionSession 加 ocr() 能力 + self-knowledge 清单登记。
- **验收**:截当前屏实测中英混排识别;错字率抽查。
- **成本**:最小;半个窗口。

## 卡④ Playwright 浏览器自动化（E 簇,社交发布链升级）
- **目标**:替换脆弱 JXA+DOM 配方,带 wait-for/截图验证的可靠网页自动化。
- **过渡方案已就绪**:MCP 配置已挂 `@playwright/mcp --browser chrome`(用系统 Chrome,免下浏览器二进制)——重启 panel 后 Noe 经 MCP 即可用,**先验证这条再决定要不要装完整 npm 包**。
- **完整步骤**(若 MCP 形态不够):`npm i -D playwright` + `npx playwright install chromium`(~300MB)→ 社交发布执行器换 Playwright driver。
- **验收**:抖音/小红书发布链在 Playwright 下走通一次带回滚证据。
- **成本**:MCP 形态零成本;完整形态 ~350MB。

## 卡⑤ session rotate 强制交接（运行态纪律原语遗留）
- **目标**:聊天房超 token 阈值时提示"轮换"→ 自动写 handoff → 新会话注入(先手动按钮不做自动)。
- **步骤**:后端 `estimateMessageTokens`(已有 NoeTrajectoryCompactor 的)算 conversation 规模 → 超阈值在 room 状态标 `rotateSuggested` → 前端 chat 头部出按钮 → 点击调 `finalizeTurn`(已建)生成交接 → 建新房注入。
- **验收**:长对话实测按钮出现→交接→新房上下文连续。
- **成本**:前后端各半个窗口;复用本窗已建的 TurnFinalizer/Compactor。
