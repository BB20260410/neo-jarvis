## 0. Noe 架构速览（强化方向从这里推导）

Noe = 本地优先的「多 AI 个人操作系统 / Jarvis 式智能体」。Node.js(Express+WS) 后端 + Web 前端，本地 127.0.0.1 不出网，可打包 Electron。版本 2.1.0，后端 45 个模块。核心依赖极精简（express/ws/better-sqlite3/@modelcontextprotocol/sdk/node-pty）——**语音/视觉/LLM 全靠外接开源项目集成**，这正是强化点最密集处。

| Noe 现有能力 | 对应模块 | 可强化方向（→ 本清单分区） |
|---|---|---|
| 语音输入(whisper) | `src/voice` `scripts/noe-whisper-server.py` | STT/VAD/唤醒词（§4） |
| 语音输出(MiniMax TTS) | `src/voice` | 本地 TTS/声音克隆（§5） |
| 视觉(qwen-vl 看屏) | `src/vision` | VLM/OCR/屏幕理解（§9） |
| 记忆(SQLite向量+图谱+RRF) | `src/memory` `src/embeddings` `src/knowledge` | 记忆/向量/RAG（§7）、知识库（§12） |
| 多模型协作 | `src/room`(50文件) | LLM 网关/路由（§10）、Agent 框架（§2） |
| 自主执行(shell/文件/权限) | `src/loop` `src/autopilot` `src/runtime` `src/permissions` | computer-use（§8）、Agent 框架（§2） |
| MCP 工具调用 | `src/mcp` `src/plugin` `src/capabilities` | MCP 生态（§3） |
| 多渠道入站/主动陪伴 | `src/channels` `src/webhook` | IM/通知/自动化（§11） |
| 整体形态参考 | — | 同类开源 Jarvis/助理（§1） |
