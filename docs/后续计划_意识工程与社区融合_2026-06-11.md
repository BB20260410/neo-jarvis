- 委托桥已经接入：owner 对话里的“帮我查/找原因/排查”能立 owner 目标。

- 社区搜索已转成正式矩阵：借工程方法，不迁 Python/大框架，不绕过安全门。

### 不采纳或暂缓

- 不直接按草稿拉 `qwen2.5:32b` 替代当前模型；模型选择以 `doctor:noe:models`、LM Studio/Ollama 实测和工具调用评测为准。
- 不直接新建 launchd plist 或改系统级调度；live `51835` 操作必须先确认端口归属并按授权执行。
- 不让内心独白直接触发行动；必须经过目标系统、显式 act 声明、ActPipeline、审批/审计。
- 不用 OpenMeow 直接替换现有 Sherpa/Kokoro/CosyVoice；只做可选 OpenAI-compatible voice gateway。
- 不整体迁移到 LangGraph/CrewAI/AutoGen/Dify；只吸收 checkpoint/resume、状态机、role/task API 等工程模式。

### 需要真实外部条件，不能伪装完成

- 真耳语音验收：真实浏览器/扬声器环境听 10 轮。
- live `51835` 是否带上最新代码：需要授权重启或确认已重启。

- 7 天自主性趋势、Brier/期望结算、开口回应率趋势。

### 来源依据

### 目标

让“去找原因/排查本地问题”成为 Noe 的产品级闭环：立项、只读诊断、证据归因、目标卡展示、Activity 可审计。

### 具体任务

1. **诊断模板细分**
   - 文件：`src/runtime/NoeDelegationExtractor.js`
   - 现状：按关键词生成通用 `rg`。
   - 改法：拆成 `voice/model/memory/goal/panel/routes` 模板，每个模板限定搜索目录和 pattern。
   - 验收：新增用例覆盖“语音断声”“本地模型没反应”“记忆没保存”“目标卡住”“面板 500”。

2. **Act 证据进入 Activity**
   - 文件：`src/cognition/NoeWorkspace.js`、`src/audit/ActivityLog.js` 或现有 activity append 入口。
   - 改法：act 完成后将 action、exitCode、stdout 摘要、stderr 摘要、goalId、stepIndex 写入 activity。
   - 验收：单测断言 activity 包含 `goalId` 与脱敏后的 stdout，不含 secret。

3. **目标卡证据展示**
   - 文件：目标/透视页相关 web 模块，先 `rg "noe_goals|goal card|目标" public/src/web src/server/routes`.

4. **安全边界**
   - 所有默认诊断命令只读。

## 4. P2 目标执行状态机

### 来源依据

- 社区参考：LangGraph checkpoint/resume、Hermes cron/observe-only、OpenHands sandbox feedback。
- 当前缺口：goal step 有 `open/doing/done`，但 waiting/blocked/recovered 语义不够完整；act/research 证据链还不够产品化。

### 目标

目标执行从“下一步被工作区推进”升级为“可恢复、可审计、可解释的状态机”。

### 具体任务

1. **扩展 step 状态**
   - 文件：`src/cognition/NoeGoalSystem.js`
   - 增加或规范：`awaiting_approval`、`blocked`、`failed`、`recovered`。
   - 保持兼容：旧 plan 里的 `open/doing/done` 不迁移破坏。

2. **Checkpoint 结构**
   - 文件：`src/cognition/NoeWorkspace.js`
   - 每次 research/act start/done 记录：goalId、stepIndex、input、action/research query、policy decision、output digest。
   - 不保存完整 stdout 大块，保留摘要和 artifact/ref。

3. **恢复策略**
   - doing 超过阈值：标记 `recovered` 或 `failed`，不自动重放危险命令。
   - approval 等待：保持 `awaiting_approval`，不重复生成审批。
   - blocked：允许后续 think 步归因，而不是无限重试。

4. **目标完成审计**
   - 每个 done goal 生成一条 summary：做了什么、证据、失败/跳过项、是否需要 owner。

### 验收标准

- 后台 act/research 异常不会让目标永久卡 `doing`。
- 审批等待不重复建单。
- 完成目标能追到每一步证据。

### 验收命令

- `npm test -- tests/unit/noe-goal-system.test.js tests/unit/noe-workspace-goals.test.js tests/unit/noe-act-pipeline-policy.test.js`
- `npm run test:p0:unit`
