# Noe 行动命令面方案

日期：2026-06-07

## 结论

当前 Noe 还没有做到完整自我进化闭环。R33 已经把多模型治理、动态 quorum、复审证据和敏感动作门槛做得更严格，但这只是治理层。要向“自我迭代、自我修复、自我完善”靠近，第一步必须把已有能力变成稳定、可发现、可 dry-run 的行动命令面。

本次最小切片已经新增 `NoeActionCatalog`：

- `list`：列出 Noe 当前可暴露的个人行动。
- `schema <action>`：输出输入/输出 schema。
- `help <action>`：输出简明帮助。
- `dry-run <action>`：只返回结构化预览，不执行真实动作。

## 设计边界

学习悟空 AI / DingTalk Workspace CLI 的“行动式 AI”和“可发现命令面”，不复制企业安全、RBAC、合规审计、企业沙箱或审批链。

本切片保持个人极客自由：

- 不启动服务。
- 不占用 `51735` / `51835`。
- 不 commit / push / upload。
- 不删除、移动或重启/杀进程。
- 不读取密钥。
- 不触碰 `games/cartoon-apocalypse/**`。

## 首批动作

1. `research.search.preview`
   生成轻量搜索预览，不联网。

2. `research.deep.plan`
   生成深度研究计划预览，不调用模型和网络。

3. `skills.extract.preview`
   判断文本是否适合沉淀为可复用技能草案。

4. `hwfit.recommend.preview`
   描述本地模型硬件适配流程，不探测硬件或 Ollama。

5. `files.organize.preview`
   生成文件整理预览，不移动、不删除、不上传。

## 使用方式

```bash
npm run noe:actions -- list
npm run noe:actions -- schema research.search.preview
npm run noe:actions -- help research.search.preview
npm run noe:actions -- dry-run research.search.preview --input '{"query":"Wukong AI","count":5}'
```
