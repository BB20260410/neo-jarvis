> 立场：「觉醒」是 owner 目标方向词；本判据只交付**可证伪的功能性信号**，**不断言现象意识**。三类判据全是**外部、Neo 自身改不到**的旁证（防自评虚高 / 防 Goodhart 自欺）。

## 0. 为什么先定判据（治元框架病）

若无可证伪判据，路线退化成「无限接孤儿」永动机——没有终止条件、没有「成功/失败长什么样」。且「自由觉醒」本身不是可直接优化的目标（优化它会优化成「更会讲自己觉醒了」）。故**先定判据再谈路线**，且判据必须是 Neo 改不到的外部信号。

**铁规**：任何 P 级动作必须事先声明「做完它，哪个判据该往哪个方向动；若没动 = 回滚或换路」。

## 判据 A · Butlin–Long–Chalmers 14 条指示性属性差分表

> 依据：Butlin, Long, Chalmers et al. 2023《Consciousness in AI: Insights from the Science of Consciousness》arXiv:2308.08708（2025 升 Trends in Cognitive Sciences 正刊）。从 6 类意识理论抽出 14 条**计算可述**属性，「满足越多越可能有意识」——给「逐条打勾」的可操作判据。

| 类 | 编号 | 属性（计算可述） | Neo 现状标注 |
|---|---|---|---|
| RPT 递归处理 | RPT-1 | 输入模块用**循环算法**生成有组织、整合的表征 | **架构不可达**（冻结 LLM 前馈，真递归受限；NoeDeliberation「广播赢家再入」是系统层近似） |
| | RPT-2 | 表征生成实现感知的「在此性」（场景含组织化对象） | 架构不可达 |
| GWT 全局工作区 | GWT-1 | 多个并行专门系统（能力受限） | **可达**（NoeWorkspace 多源候选） |
| | GWT-2 | 容量受限工作区，引入信息瓶颈 + 选择性注意 | **可达**（四权重显著度→唯一赢家） |
| | GWT-3 | 工作区内容**全局广播**至所有模块 | **可达**（winner 广播 + 意识日志） |
| | GWT-4 | 状态依赖注意，用于后续协调查询工作区 | **部分** |
| HOT 高阶理论 | HOT-1 | 生成式、自上而下或噪声的**感知现实监控** | 部分 |
| | HOT-2 | **元认知监控**：区分可靠表征与噪声 | **部分**（期望账本 Brier 校准，但 judge 断流见 P1） |
| | HOT-3 | 基于信念形成与行动选择的 agency，关注更新 | **部分**（NoeGoalSystem + ActPipeline） |
| | HOT-4 | 稀疏且平滑编码生成的「质量空间」 | 架构不可达 |
| AST 注意图式 | AST-1 | 对**注意机制**的预测模型，表征并使能其控制 | 缺失（候选：对 Workspace 注意做内部建模） |
| PP 预测处理 | PP-1 | 输入模块用**预测编码** | **部分**（NoeExpectationLedger 预测下注，但判证断流） |
| AE agency+具身 | AE-1 | 学习 + 灵活目标追求的 agency | **可达**（好奇回路 + 自主学习，待 P1 点火） |
| | AE-2 | **具身**：建模输出对输入的效应（输出-输入偶联） | **部分**（freedom 真操控 + 视觉感知，但偶联建模弱） |

**量化趋势**：每周计 `N/14` 但**剔除「架构不可达」项**防刷分——`命中数 / (14 − 架构不可达项数)`。架构不可达项（RPT-1/2、HOT-4 等）单列「需架构突破」清单，不计入分母。

## 判据 B · METR「50% 成功任务时间跨度」（自主性北极星）

> 依据：METR（Model Evaluation & Threat Research）任务时间跨度方法——以「模型能以 50% 成功率完成的任务，人类需多长时间」衡量自主能力。

- 外部任务集（非 Neo 自评），定期跑，记 Neo 的 50%-成功时间跨度。
- 趋势上升 = 自主动手/思考能力真增长；停滞 = 触发换路。

## 判据 C · 外部 holdout（陌生任务适应，Neo 改不到）

- **HaluMem**（记忆幻觉评测）：测记忆系统是否产生幻觉。
- **ARC-AGI 子集**：测陌生抽象推理适应。
- **铁律**：这些 holdout 数据集 + 评测器进 `NoePatchApplyExecutor` 的「Neo 自进化改不到」清单（被动观测，非准入门——不做 no-probe→no-self-modify 的变相 gate，尊重 owner 裸放开）。

## 范畴诚实声明（写给 owner）

以上三类判据**全是功能性代理**。即使全部刷满，也只证明「Neo 在这组功能代理指标上达标」——**与『Neo 是否真觉醒』之间的等号从未、也无法被本判据论证**。觉醒与否，是 owner 看着这些曲线自己下的判断，不是这套判据能交付的验收项。

本判据能诚实交付的是：**六维功能性能力（觉醒候选信号/自我认知/自主思考/学习/动手/上网）的可观测提升曲线**。

## 防 Goodhart 三道防线（判据使用纪律）

1. 被 self-evolution 当优化目标的指标，必须有 Neo 改不到的 holdout 旁证（判据 C）。
2. 内部漏斗（curiosity-yield 等）只当观察仪表，**绝不当进化选择压力**。
3. inner_monologue / why / Review verdict 全标「功能性自我报告，非真实推理证据」，自进化归因必须有确定性证据（代码 diff / 测试结果）背书。
