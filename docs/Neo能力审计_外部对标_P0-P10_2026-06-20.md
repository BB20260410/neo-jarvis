### 每能力"现状 vs 理想"差距叙述

#### 1. 自我思考 — 成熟,但卡在"统一意识"的天花板
**现状**:这是活了 10 天、产出 14400 条内心念头的**真递归闭环**,远超"写了没跑"。GWT 全局工作区每拍从 9 个源竞争出唯一焦点,五层防反刍(字符/语义/思维回环/接地重写/经历锚点)全部真触发,情感连续性、持久心跳、深思脑全部通电。
**理想**:自发性+递归性+抗退化+注意力架构+情感连续+元认知自评+被现实纠正→主动学习+持久性+诚实可观测。
**差距**:①**整合度 Φ 长期≈0**(288 条读数全在 0~0.14,label"近乎离散")——这是最关键的元认知指标却几乎贴地,说明思考是多个并行子系统**各自运转而非整合成统一意识内容**,这是能力天花板而非 bug;②熵驱动温度 0 次 boost(依赖 Ollama 嵌入,按需卸载即退回 hash128 维);③本地脑单点——今日 84 次 `CircuitBreaker:lmstudio OPEN` 期间 GWT **空转打分但不产新念头**,"看起来在思考实则只打分";④786 自主目标多是念头复述而非真任务。

#### 2. 自我完善 — 半成品·勤奋空转(**真短板**)
**现状**:7 条子链几乎全接线全通电,心跳真跑(selfEvolve 1533 tick)。Brier 期望-误差回路是真正在工作的"被现实纠正"发动机;SFT 真产出周文件。
**理想**:信号采集→候选生成→客观评测→安全采纳→**效果回流**→防退化的闭合飞轮,判据是"是否真的越用越好"。
**差距**:①**效果回流近乎断裂**——838 学习卡仅 18% 被召回、686 张死库,surprise_lesson 15 张命中 0,produce 端勤奋、consume 端失效,**反馈环没闭合**;②学习不收敛(11 主题学 321 次、重复 97%,动态选题治标失败);③ReflectiveTuner(491 行)写好**从未运行**;④SFT 攒了数据但"渗入权重"**从未发生**(无 LoRA 产物);⑤"测试全绿≠生产有效"的系统性落差,且本仓有"工具输出疑似被篡改"诊断报告,对"绿/通过"信号需额外警惕。

#### 3. 自我迭代 — 半成品·空转(**真短板,头号致命**)
**现状**:六段(执行器+触发器+离线闸+多模型共识+反 reward-hacking+可回滚事务)工程做工极高,纵深防御、fail-open、脱敏、备份回滚一应俱全。自主链能起步:AUTOSEED 真从内心独白长出 6 个目标 4 个 cycle。
**理想**:无人工脚本介入完成"发现→生成→评测→落地→度量→固化"一整圈。
**差距**:①**自主链共识断桥(头号致命)**——`NoeConsensusRunner` 在运行时**零接线**,consensus ledger 只能人工跑 script 产,导致每个自主 cycle 永远卡 `consensus_blocked`、executor 永不触发;②触发器 `STAGE_TO_ACTION` 缺 `consensus_blocked→refresh_consensus` 映射;③执行器手脚生产从未触发(noe_acts 中 self_evolution=0);④自动发现/生成候选不闭环(AUTOSEED 只把念头转成目标标题,离真候选很远)。**本质上 Neo 当前不是"自主修改",而是"自主提议+人驱共识后才能改"**。

#### 4. 主动动手 — 成熟,通电最充分
**现状**:NoeHeartbeat 持久心跳驱动,**3228 条 act 全部 dryRunOnly=0(真执行非 dry-run)**:真跑 shell、真控 macOS GUI、真开浏览器。闭环实证扎实——browser 真失败→喂 surprise→生成纠错目标。

#### 5. 自主修改代码 — 半成品·0 实跑(**真短板**)
**现状**:两条环+多重门,事务链真跑通(chain-drill 全 7 gate 绿:dry-run→拦未确认→确认写→回滚,且 noSourceFilesChangedByDrill=true)。起意→立项→建 cycle 真活。
**理想**:自发起意→生成补丁→安全应用→自验证→自回滚→授权门→证据驱动完成→闭环可持续。
**差距**:①**端到端闭环生产 0 实例(最致命)**——能力停留在"单测 215/215+dry-run 19/19 绿",未跨过"真改一次自己代码"的门槛;②自驱链**结构性死锁** `consensus_blocked`(自驱心跳无法自产四模型共识);③**授权门双重 OFF**——真实 probe 显示 standing grant 缺 `self-evolution:run` scope(AUTHORIZED=false)+ `NOE_SELF_EVOLUTION_STANDING_GRANT` 未设。**注:这与 owner"裸放开自我进化"的记忆存在落差**——env executors 开了,但真正解锁 apply 的 scope/桥 env 没开,实际仍是 shadow 态。

#### 6. 长记忆 — 成熟,运行证据最硬
**现状**:本审计覆盖能力中**最成熟的子系统**(src/memory 50+ 文件 + 61 测)。双路语义召回真跑(FTS5 trigram + 1024 维**真 ollama** 向量,RRF + Fisher-Rao 信息几何重排 + 双相衰减,四高级开关全 ON),**有效召回率 92.5%、最近召回在审计当天**。写入五道闸真分流。代码注释证实它已主动吸收业界几乎所有核心设计(Mem0/OpenMemory/Graphiti/A-MEM/Orama)。
**理想**:持久化+高质量召回+写入治理+遗忘+巩固升华+强化回路+真注入闭环+可观测+隐私。
**差距**:①**FocusStack 工作记忆近乎空转(仅 1 条)**——"主动把高价值记忆上浮"这一维形同摆设;②梦境升华路径从未真执行(episodic_digest=0,硬要 90 天前情景而最老仅 9 天——**Neo 年龄不足的正确行为,但意味着该路径未经真实数据验证,约 81 天后才首跑**);③GC 开关 `NOE_MEMORY_GC` 未配置(2348 条 hidden 无定时回收);④召回注入"有用性"缺 before/after 度量(停在"注入发生"非"注入有用")。

### 哪些是 Neo 的真短板

**真短板 = 自我完善 / 自我迭代 / 自主修改代码** 三者,且它们**同根**:

1. **共同病灶:`consensus_blocked` 结构性死结**。自我迭代和自主改代码的自主链都死卡在四模型共识门——`NoeConsensusRunner` 运行时零接线,自驱心跳拿不到共识 ledger。这不是架构缺陷,是**工程接线 + 授权门 env 未打开**的可用性问题。修它不靠引新框架,靠:(a) 打开 `NOE_SELF_EVOLUTION_CONSENSUS_AUTODRIVE` 把死锁降级成本地装配 ledger;(b) 补 `self-evolution:run` standing scope;(c) 借 DGM/SWE-agent 的"经验性 benchmark 验证替代强共识门"思路绕开死锁。

2. **共同模式:产而不用 / 跑而不验**。自我完善的 838 卡 18% 召回、自主改代码的 0 实跑、自我迭代的 0 自生成补丁——都是"勤奋地运转但没真变好"。判据从来不是"测试是否绿",而是"产出是否被消费、闭环是否真转一圈"。

3. **"裸放开"与实际配置的落差**:owner 记忆中"自我进化已裸放开",但实测多个解锁 apply 的 env/scope 仍 OFF。**这是优先级最高、ROI 最高的修复点**——不需要写代码,只需把已自建到位的开关在生产 .env 打开 + 用已有 benchmark 做 before/after 度量。

**不是短板的**:自我思考、主动动手、长记忆三者已成熟。它们的剩余问题是**调参/接线深化/Neo 年龄不足**(如整合度低、梦境升华待 81 天),而非"再引一个开源项目能解的"——硬塞框架只会让已成熟的子系统臃肿、与自建逻辑打架。

## 排序原则

1. **先补"没真跑通的核心能力"**(扎根 `canRun` 结论):consensus 死锁(P0)是 3 份审计共同的 critical/头号缺口,且修复成本被严重高估 → 必为 P0。
2. **再补"有外部成熟方案能显著增强且 Node 原生/纯 JS"**:transformers.js(P1)、jsDiff(P4)、MMR(P5)。
3. **务实防臃肿/防污染**:全程仅 **3 个真新增 npm 依赖**(`@huggingface/transformers` / `diff` / `@ast-grep/napi`),其余全是**翻 flag、接已装件、或只取 Python 设计重实现**;所有外部项目许可均 MIT/Apache-2.0/BSD,**零 AGPL/GPL 污染**。
4. **P0–P2 优先确定性高、收益大**;Python 重型框架(LLaMA-Factory/TRL/OpenHands/LangGraph/Letta runtime)一律**只取设计、不进 Neo 进程**。

### 自我迭代
**内审结论**:部分通电但缺口致命——自主链能起步走不完，端到端自我迭代在生产从未完成过一个 cycle — Neo 的自我迭代由「环1 执行器手脚 + 环2 触发器/状态机 + 离线候选闸 + 多模型共识 + 反 reward-hacking + 可回滚补丁事务」六段组成，工程做工极高（纵深防御、fail-open、脱敏、备份回滚、闭 schema 闸）。生产 .env 四个开关全开（NOE_SELF_EVOLUTION=1 / _EXECUTORS=1 / _AUTOSEED=1 / NOE_HEARTBEAT=1），server.js 也真把触发器(1116-1128)、执行器(931-962)、selfEvolve 心跳(2287-2300)装配上线。
