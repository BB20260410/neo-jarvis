## 解决的缺口
此前 Neo 的「该研究什么」**只有静态预定义主题表**（`NoeLearningTopics.js` 的 `NOE_LEARNING_TOPICS` 6 条 + `NOE_LEARNING_CONCEPTS` 24 条，全人工列死）。`NoeTopicCurator` 的饱和冷却也只是「在这张静态表里换着选」，Neo 不能从自身状态自主长出新的研究主题种子。

本模块从三个真实状态源动态发现主题种子：
1. **记忆里反复出现却没深究的实体**（源 `unexplored_entity`）：读 `NoeKnowledgeGraph.search` 的 `noe_kg_entity`，`mention_count` 高（反复出现）+ `description` 空/极短（没深究）= 知识缺口。
2. **好奇回路高好奇信号**（源 `curiosity_signal`）：读 `NoeGoalSystem.list({source:'surprise'})`，`meta.curiosity.score`（`NoeCuriosityDecompose` 双因子分）高的 → 抽 claim 成主题。
3. **owner 未兑现承诺 / 开放回路**（源 `open_commitment`）：读 `NoeCommitmentStore.list({status:'open'})` 的 `open_loop`/`task` 类（`reminder`/`event_check_in` 日程类排除）。

产物形状 `{title, url, query, source, evidence, score}`，其中 `{title,url,query}` 子集与 `NoeTopicCurator.getNextTopic` 的 `dynamicConcepts` 入参兼容。

## 种子质量与去重
- **质量闸**（借鉴 P0-4 AUTOSEED 精神，`isResearchableSeed`）：种子须含可定位技术对象（英文标识符/驼峰/版本号/url）**或**明确问句（为什么/怎么/how/why…），且不是纯情绪寒暄碎片、不是纯泛词。情绪碎片（"谢谢"/"加油"）、太短、纯泛词问句（"这个东西是什么"）一律拒。
- **去重**三层：① 与静态表 `NOE_LEARNING_CONCEPTS` 精确（url/标题）+ 名级（同概念不同 query 串，如静态表 `?q=Letta+MemGPT` vs 实体源 `?q=Letta`）兜底；② 与 `NoeTopicCurator` 访问账本 `noe.learning.topicArchive.v1`（近期已研究）；③ 源间互相（同 url 高分优先留一条）。
- **fail-open**：任一数据源缺失/抛异常 → 该源贡献 0 种子，绝不抛穿、绝不污染。

## .env 开关（默认 OFF）
```
# P10 发现端：动态发现研究主题（从记忆未深究实体 / 好奇信号 / owner 开放回路）。
# 默认 OFF（分量动作：自主发现会驱动自主研究）。=1 开启。
NOE_TOPIC_DISCOVERY=1
```
`src/bootstrap/load-env.js` 在 import 期把 `.env` 装进 `process.env`，此 flag 自动被 `createTopicDiscovery` 的 env 默认读到。

## 点火步骤（owner 在场时）
1. **接线（尚未做，留给装配窗）**：在 `noe.js` / server 装配处，用真实依赖建实例：
   ```js
   import { createTopicDiscovery } from './src/cognition/NoeTopicDiscovery.js';
   import { collectLearningConcepts } from './src/cognition/NoeLearningTopics.js';
   const topicDiscovery = createTopicDiscovery({
     kg,                 // NoeKnowledgeGraph 实例（只读 search）
     goalSystem,         // NoeGoalSystem 实例（只读 list）
     commitmentStore,    // NoeCommitmentStore 实例（只读 list）
     kv,                 // 取 topicArchive 做近期已研究去重
     staticConcepts: collectLearningConcepts(),
   });
   ```
   然后把 `topicDiscovery.discoverConcepts()` 的产物**并入** `NoeGoalSystem.maybeSeedAutonomousLearning` 喂给 `topicCurator.getNextTopic({ dynamicConcepts })` 的那一处（现仅喂 `collectLearningConcepts()`）：
   ```js
   const dynamicConcepts = [
     ...collectLearningConcepts(),
     ...(topicDiscovery.enabled ? topicDiscovery.discoverConcepts() : []),
   ];
   ```
   注意：需要 `NOE_DYNAMIC_TOPICS=1`（动态选题管道）+ `NOE_AUTONOMOUS_LEARNING=1`（自主学习循环）才真正驱动研究；`NOE_TOPIC_DISCOVERY` 只控「是否产新种子」。
2. **隔离端口验证**：`PORT=51999 NOE_TOPIC_DISCOVERY=1 NOE_DYNAMIC_TOPICS=1 npm run start:noe`，看自主学习目标的 research 步是否开始研究**库里没列过**的主题（而非永远那几个静态页）。
3. **回收**：关 `NOE_TOPIC_DISCOVERY` 即回到「只用静态表」零回归。
