## 3. 5 个路线详细内容(每个 5-10 行)

### 3.1 P6-7 · 自演化循环提速(7-10 天)

**目标**:owner 自由立场下彻底放开自进化频率
- `NOE_SELF_EVOLUTION_OBSERVE_COOLDOWN_MS` 30min → **5min**(env 注入)
- 心跳拍 60s → **30s**(6 处 `Math.max` 下限放宽)
- `NOE_SELF_EVOLUTION_REAL_APPLY` 默认仍 OFF(dry-run 安全)
- 借鉴 AutoGPT continuous_mode(立刻落地 1 行 log)

**关键文件改动**:
- `server.js:1149-1163` 自进化 trigger 装配段
- `src/room/NoeSelfEvolutionTrigger.js:9` cooldown 常量
- `.env` 默认值

**红线遵守**:不加 owner 审批、不重做 P0-P5、不动 PolicyFileGuard

### 3.2 P6-9 · Neo 自我讲故事 / 反思日记(2 周)

**目标**:Neo 每周自动写周记,跨周可对比
- 触发:`NOE_SELF_NARRATIVE_WEEKLY=1`(默认 ON),每周一次
- 数据源:5 个(本周 cycle / acts / memory 写入 / GWT metrics / VAD 平均 / lessons)
- 输出:结构化 JSON 周记 → 写入 `noe-self-narrative` 记忆
- 借鉴 generative_agents Memory Stream reflection

**关键发现**:Neo 已有 NoeNarrativeSelf + NoeNightlyReflection,P6-9 补**周级量化 + 跨周对比**,**互补不重做**

**关键文件改动**:
- 新增 `src/cognition/NoeSelfNarrative.js`(周记生成器)
- 新增 `src/server/services/noe-self-narrative-scheduler.js`
- 改造 `src/server/services/noe-maintenance.js`(装配点)

**周记结构**:周期 + stats + highlights + cognitive_state + lessons_learned + next_week_focus

### 3.3 P6-6 · 真人陪伴体验层(12-15 工作日)

**目标**:把 VAD / Focus Stack / GWT 可视化
- **头像 + 情感色彩**:HSL 色相随 (v × 360)、亮度随 arousal、饱和度随 dominance
- **思维气泡**:实时显示 Focus Stack 顶部 3 帧 + GWT 赢家候选
- **TTS 情感色彩**:语速随 arousal、音调随 valence、停顿随 dominance

**关键发现**:后端 4 件(VAD / Focus Stack / GWT / Inner State Provider)已就位,**前端 0 接点**

**API 设计**:`GET /api/companion/state` 返回 VAD + Focus Stack + GWT 赢家 + TTS 情绪参数

**借鉴**:moshi(端到端实时) + CosyVoice(中文流式 TTS) + screenpipe(屏幕保护模式)

**关键文件改动**:
- 新增 `public/companion/`(avatar.js / thinking-bubble.js / tts-color.js / companion.css)
- 新增 `src/server/routes/noe-companion-state.js`
- 改造 `public/index.html` + `public/app.js` + `src/context/NoeInnerStateProvider.js` + `src/voice/VoiceSession.js`

### 3.4 P6-3 · 跨进程协作(8-12 周单人 / 4-6 周 2 人)

**目标**:拆 server.js 3130 行 monolith 为 6 独立 Node 进程
- `chat-server`(主入口)+ `memory-server` + `watcher-server` + `voice-server` + `vision-server` + `self-evolution-server`

**IPC 协议**:JSON-RPC 2.0 + Unix Domain Socket(本机)+ WebSocket(跨机)

**实施分阶段**:
- 阶段 1:拆 1 个(memory-server,2-3 周)— 验证 IPC 模式
- 阶段 2:拆 2 个(watcher + voice,4-6 周)
- 阶段 3:拆 3 个(vision + self-evolution + chat,4-6 周)

**关键文件改动**:
- 新增 6 个 server(chat-server / memory-server / watcher-server / voice-server / vision-server / self-evolution-server)
- 新增 `src/ipc/` 目录(json-rpc.js / unix-socket.js / websocket.js)
- 大量改造现有 src/ 模块(in-process → IPC)
- 改造 `package.json` + `electron-main.js`

**借鉴**:openai/swarm(极简 handoff)+ MetaGPT(SOP 角色调度)+ AgentVerse(多 agent 编排)

**风险**:IPC 延迟 / 分布式调试 / 共享 state / IPC 鉴权 / 部署复杂性

### 3.5 P6-1 · 完整 Jarvis UI(4-8 周)

**目标**:对话优先 + 卡片式 + PWA-ready

**3 个核心改造**:
- **对话优先布局**:主区对话流 / 侧栏 Brain 状态 / 顶栏 Companion 头像 / 底栏输入框
- **卡片式消息**:每条独立卡片,可拖拽 / 折叠 / 固定 / Markdown
- **PWA-ready**:manifest.json + Service Worker + iOS add to home screen

**关键发现**(来自 worker 的重要校正):
> **Neo 不是 731 行工程师控制台,而是 88 模块 + 30 CSS + 55 modal 的高复杂度面板**。组件化已完成 80%,**真正缺的是对话流布局 + 卡片化消费后端 ACUI + PWA**。

**借鉴**:lobe-chat(UI 设计) + Flowise(拖拽工作流) + langflow(visual agent) + chatbot-ui(ChatGPT 风格)

**7 种卡片类型**:user / assistant / system / tool 调用 / memory 调用 / code block / error

**实施分阶段**:
- 阶段 1:重构消息卡片(2-3 周)
- 阶段 2:重构侧栏 + 顶栏(2-3 周)
- 阶段 3:PWA 配置 + manifest + Service Worker(1-2 周)
