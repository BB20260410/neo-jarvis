> 来源仓库 [github.com/pewdiepie-archdaemon/odysseus](https://github.com/pewdiepie-archdaemon/odysseus)（dev 分支，**MIT**，50.1k★，self-hosted AI workspace，Python 后端 + 原生 JS 前端）

## 〇、技术现实（定调）

**Noe 是 Node/JS，Odysseus 后端是 Python。** 所以"移植"= 移植**设计/算法**用 JS 重实现，或复用其原生 JS 前端 —— 不复制 Python 文件（塞进去跑不了又臃肿）。下面所有"推荐"均按此口径。

## 一、分级总表

| 模块 | Odysseus 路径 | 评级 | 一句话理由 |
|---|---|---|---|
| **多步研究 DeepResearcher** | `src/deep_research.py` `services/research/` | ⭐**强烈推荐** | Noe 能力空白，补"上网研究"硬缺口，与协作体系正交无重叠 |
| **网络搜索 (SearXNG 链路)** | `services/search/` `config/searxng/` | **推荐** | 免 key 自托管搜索，是上面 Research 的基础设施；补能力⑤ |
| **硬件扫描荐模型 hwfit** | `services/hwfit/` | **推荐** | 算法轻量(sysctl+量化降级)；但静态模型库会过时，只移植逻辑不搬数据 |
| **Skills 自动提取** | `services/memory/skill_extractor.py` | **推荐** | Noe 真实缺口（现只能手动建 skill），轻量 ~120 行 |
| RAG 工具筛选 | `src/tool_index.py` | 有条件(低优先) | 仅当 MCP 工具数 >30 才值得，现在不急 |
| Document 编辑器 | `services/docs/` `routes/document_*` | ❌**不推荐** | 实测**无 AI 续写/改写**功能；PDF 部分 PyMuPDF=AGPL；Noe 定位不需编辑器 UI |
| Agent loop / MCP manager | `src/agent_loop.py` `src/mcp_manager.py` | ❌不推荐 | Noe 已有 AgentRunLifecycle + McpClientManager，完全重复 |
| Memory 系统 | `services/memory/memory*.py` | ❌不推荐 | Noe 的 SQLite+FTS5 强于 Odysseus 的 JSON+ChromaDB，移植=降级 |
| Skills 存储+注入 | `services/memory/skills.py` | ❌不推荐 | 与 Noe SkillStore/skillInjector 格式完全相同 |

## 二、强烈推荐 · DeepResearcher 多步研究

**核心编排**（值得移植的是这个设计，不是代码）：
```
plan → [最多 8 轮 / 300 秒双上限]:
  LLM 生成查询 → 并行搜索(多源故障转移) → 并行抓 URL
  → goal_based_extractor 结构化提取 {rational, evidence, summary}
  → LLM 综合进 evolving_report(渐进演化,非最后一次性生成)
  → LLM 判断"是否足够"(连续2轮空结果自动停)
→ 最终 LLM 润色报告
```
**亮点**：`evolving_report` 渐进综合、`goal_based_extractor` 结构化提取（而非全文暴力塞入）、`queries_used/urls_fetched` 跨轮去重、双上限防失控。
**与 Noe 关系**：Noe 的 5 种 dispatcher 解决"多 AI 协作分工"，Research 解决"单 agent 自驱信息收集"——**正交，零重叠**。
**JS 落地**：`src/research/DeepResearcher.js` ~300-400 行，复用已有 BrainRouter（LLM 调用）+ 下面的 WebSearch（搜索层）。

## 三、推荐 · 网络搜索（SearXNG 链路）

**方案本质**：SearXNG 自托管元搜索（**100% 免 key**）为首选 + DuckDuckGo/Brave/Google/Tavily/Serper 降级链。
**查询接口**（实测源码 `services/search/providers.py`）：
```
GET {searxng}/search?q=<query>&format=json&engines=bing,mojeek,presearch&time_range=week
→ data.results[] 每条取 {title, url, content(snippet)}
```
**完整链路**：search → 本地排序 → 并行抓页(httpx+BeautifulSoup) → 分级正文提取(优先 `<main>/<article>`，回退 `<body>` 去噪) → 格式化喂 LLM。
**依赖**：Docker+SearXNG 容器(可选) / httpx→`fetch` / beautifulsoup→纯正则或 cheerio。
**协议**：Odysseus 代码 MIT；SearXNG 本体 AGPL-3.0，**但以 HTTP 客户端查询网络服务 = 不触发 copyleft**（客户端无需开源），风险低。
**国内可行性**：SearXNG 实例走 Clash 代理即可访问 Bing/Mojeek，比 DuckDuckGo 直连可靠得多。
**JS 落地（Noe 适配版）**：做成**可插拔 provider 抽象**，不强制 Docker——
- provider 链：SearXNG(`NOE_SEARXNG_URL`，可填自托管或公共实例) → Brave(`NOE_BRAVE_KEY`，免费 2000次/月) → 友好降级提示
- 网页抓取：Node `fetch` + 轻量正则正文提取（**零新依赖**，不引 cheerio，控体积）

## 四、推荐 · 硬件扫描荐模型 hwfit

源自 [llmfit](https://github.com/AlexsJones/llmfit)（MIT）。**Apple Silicon 探测仅 3 行 sysctl**（可直接抄逻辑）：
```
machdep.cpu.brand_string → 芯片名(Apple M5 Max)
hw.memsize → 统一内存总量
iogpu.wired_limit_mb → Metal 工作集(可选)
GPU 预算：>64GB→80% / 16-64GB→75% / ≤16GB→67%
量化降级阶梯：Q8_0→Q6_K→Q5_K_M→Q4_K_M→Q3_K_M→Q2_K
```
**关键判断**：它的 `hf_models.json`(475KB 静态库,截至 2026-02) **会过时,不要照搬**。Noe 应改成**实时查 Ollama**(`GET /api/tags` + `ollama.com/search`) + 量化规则引擎。
**JS 落地**：~300 行，零依赖，`child_process` 跑 sysctl。

## 五、推荐 · Skills 自动提取

Odysseus `skill_extractor.py`：会话结束后（轮次≥2 或工具调用≥2），LLM 一次调用判断能否提炼可复用技能，**置信度 <0.6 丢弃**，去重，异步非阻塞。
**Noe 缺口**：Noe 的 SkillStore 格式已兼容 Claude Skills，但**只能手动建**。这个自动提取是真实补强。
**JS 落地**：`src/skills/SkillExtractor.js` ~120 行，复用 BrainRouter(Haiku 级够用) + 现有 SkillStore.create()。

## 六、不推荐（避免臃肿，逐项说明）

- **Document 编辑器**：调研实测它**根本没有"AI 续写/改写/总结"**（只有 PDF 表单自动填写 + 垃圾文档分类）；PDF 依赖 **PyMuPDF=AGPL-3.0**（协议炸弹）；Noe 是贴身助手不是文档工具。若真要编辑器，自己嵌 CodeMirror6 比移植干净 10 倍。
- **Agent loop / MCP manager**：Noe 已有 `AgentRunLifecycle.js`+`AgentStateMachine.js`+`McpClientManager.js`，完全重复。
- **Memory 系统**：Odysseus = JSON 文件 + ChromaDB(额外进程)；Noe = SQLite+FTS5(原生全文检索,更强)。移植 = 降级。
- **Skills 存储/注入**：与 Noe `SkillStore.js`/`skillInjector.js` 机制完全相同。

## 七、落地路线图（ROI 从高到低）

1. **P1 立即**：`WebSearch` 搜索基础模块（补能力⑤ + 作 Research 地基）→ `DeepResearcher` 多步研究编排。**两者合起来 = Noe 终于能上网做研究**。
2. **P2**：`hwfit` 硬件荐模型（M5 Max 用户选模型痛点）。
3. **P3**：`SkillExtractor` 会话后自动提炼技能。
4. **可选**：MCP 工具数 >30 时再做 RAG 工具筛选。
